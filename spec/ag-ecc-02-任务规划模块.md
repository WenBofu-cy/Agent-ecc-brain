# ag-ecc-02-任务规划模块 接口规格

---

## 基本信息

| 项 | 内容 |
|----|------|
| 模块编号 | ag-ecc-02 |
| 模块名称 | 任务规划模块 |
| 所属分区 | 一、认知大脑核心模块 |
| 核心职责 | 接收 ag-ecc-01（意图解析模块）输出的结构化意图描述，将用户的高层任务目标拆解为可执行的有序步骤序列。评估每一步骤的前置依赖、资源需求与预期耗时，生成完整的任务执行计划。同时负责管理任务的全生命周期——包括任务队列的优先级排序、执行进度跟踪、异常中断后的恢复规划。支持多任务并发的调度编排。不参与具体工具的选择或执行，仅负责任务的逻辑拆解与流程编排 |
| 依赖模块 | ag-ecc-01（意图解析模块，接收结构化意图）、ag-ecc-03（工具选择模块，获取可用工具能力约束）、ag-ecc-05（记忆查询模块，查询历史任务模板）、ag-ecc-07（工作记忆模块，读写任务上下文） |
| 被依赖模块 | ag-ecc-03（工具选择模块，接收步骤需求）、ag-ecc-04（安全仲裁模块，提交计划供安全审查）、ag-ecc-06（结果评估模块，接收任务完成回执）、ag-ecc-09（内生动机模块，接收自主目标任务） |


## 内部状态定义

| 状态 | 标识 | 含义 | 触发条件 |
|------|------|------|----------|
| 等待意图 | `WAITING_INTENT` | 无待处理意图，等待 ag-ecc-01 输入 | 系统初始化完成，任务队列为空 |
| 拆解规划中 | `PLANNING` | 正在将意图拆解为可执行步骤序列 | 收到结构化意图描述 |
| 等待资源确认 | `AWAITING_RESOURCES` | 规划完成，等待工具选择模块确认资源可用性 | 步骤计划生成后，需要工具能力确认 |
| 任务执行中 | `EXECUTING` | 计划已下发，正在跟踪执行进度 | 计划通过安全仲裁并下发至执行层 |
| 异常处理中 | `ERROR_RECOVERY` | 某步骤执行失败，正在生成恢复计划 | 收到执行层的步骤失败通知 |
| 暂停服务 | `SYSTEM_PAUSED` | 系统紧急熔断 | 收到紧急熔断指令 |


## 输入数据

| 输入项 | 数据类型 | 来源模块 | 触发条件 | 优先级 |
|--------|----------|----------|----------|:---:|
| 结构化意图描述 | Struct（意图ID + 会话ID + 任务类型 + 实体列表 + 约束条件 + 优先级 + 置信度） | ag-ecc-01 意图解析模块 | 意图解析完成时 | **高** |
| 工具能力约束 | Struct（可用工具清单 + 各工具参数约束 + 调用成本 + 当前负载） | ag-ecc-03 工具选择模块 | 规划过程中按需查询 | **高** |
| 历史任务模板 | Struct（相似任务的历史步骤序列 + 成功/失败标记 + 平均耗时） | ag-ecc-05 记忆查询模块 | 规划过程中按需查询 | 普通 |
| 步骤执行回执 | Struct（步骤ID + 执行状态：成功/失败/超时 + 输出数据摘要） | ag-ecc-06 结果评估模块 | 每个步骤执行完成时 | **高** |
| 任务中断通知 | Struct（中断步骤ID + 中断原因 + 当前执行进度） | ag-ecc-06 结果评估模块 | 任务异常中断时 | **高** |
| 自主目标任务 | Struct（目标描述 + 优先级 + 生成原因） | ag-ecc-09 内生动机模块 | 系统自主发现新任务时 | 普通 |
| 全局调度指令 | Enum（暂停/恢复/熔断） | ag-ecc-12 资源调度模块 | 模式切换或紧急事件时 | **紧急** |


## 输出数据

| 输出项 | 数据类型 | 目标模块 | 输出条件 | 优先级 |
|--------|----------|----------|----------|:---:|
| 任务执行计划 | Struct（计划ID + 意图ID + 步骤序列：每步含步骤描述、预估工具类型、依赖步骤ID列表、预估耗时、失败重试策略） | ag-ecc-03（工具选择）、ag-ecc-04（安全仲裁） | 规划完成时 | **高** |
| 步骤资源需求 | Struct（步骤ID + 所需工具类别 + 参数约束 + 期望完成时间） | ag-ecc-03 工具选择模块 | 规划完成时 | **高** |
| 任务状态更新 | Struct（计划ID + 当前进度 + 已完成步骤 + 剩余步骤 + 预估剩余时间） | ag-ecc-07 工作记忆模块、用户交互层 | 步骤状态变更时 | 普通 |
| 恢复计划 | Struct（原计划ID + 失败步骤ID + 替代步骤序列 + 回滚步骤序列） | ag-ecc-06 结果评估模块 | 任务中断时 | **高** |
| 规划状态上报 | Struct（当前状态 + 活跃任务数 + 队列深度 + 平均规划耗时） | ag-ecc-12 资源调度模块 | 周期性或状态变更时 | 普通 |


## 任务拆解规则

### 一、拆解策略矩阵

| 任务类型 | 拆解策略 | 典型步骤模式 | 并行可行性 |
|----------|------|------|:---:|
| 信息查询 | 串行逐步细化 | ①确定查询源 → ②执行查询 → ③整理结果 | 多源查询可并行 |
| 工具调用 | 依赖驱动串行 | ①参数准备 → ②工具调用 → ③结果校验 → ④（可选）后续处理 | 步骤严格串行 |
| 内容创作 | 阶段式流水线 | ①需求澄清 → ②大纲规划 → ③内容生成 → ④润色排版 | 可流水线重叠 |
| 对话交互 | 直接单步 | 生成回复 | 无需拆解 |
| 任务管理 | 查找→操作→确认 | ①查找目标任务 → ②执行管理操作 → ③确认结果 | 串行 |
| 系统配置 | 校验→设置→验证 | ①校验当前配置 → ②写入新配置 → ③验证生效 | 串行 |

### 二、优先级排序规则

| 排序因子 | 权重 | 说明 |
|----------|:---:|------|
| 用户显式优先级 | 最高 | 用户明确标注为"紧急"的任务优先 |
| 任务等待时长 | 中 | 等待时间越久越优先，防止饥饿 |
| 预估耗时 | 低 | 短任务优先，提升整体吞吐量 |
| 依赖关系 | 约束 | 被其他任务依赖的任务自动提升优先级 |


## 核心处理逻辑

```
FUNCTION task_planner_main_loop():
    STATE_WAIT = WAITING_INTENT
    STATE_PLAN = PLANNING
    STATE_AWAIT_RES = AWAITING_RESOURCES
    STATE_EXEC = EXECUTING
    STATE_ERR = ERROR_RECOVERY
    STATE_PAUSED = SYSTEM_PAUSED

    SET internal_state = STATE_WAIT
    初始化任务队列 = []
    初始化活跃任务表 = {}  // 计划ID → 任务状态

    WHILE 系统运行中:
        // 第1步：紧急熔断
        IF 收到紧急熔断指令:
            SET internal_state = STATE_PAUSED
            CONTINUE
        ELSE IF 收到恢复指令 AND internal_state == STATE_PAUSED:
            SET internal_state = STATE_WAIT

        // 第2步：接收新意图
        IF 收到 ag-ecc-01 结构化意图描述:
            SET internal_state = STATE_PLAN
            意图 = 意图描述
            计划ID = 生成计划ID()

            // 2a. 根据任务类型选择拆解策略
            拆解策略 = 获取拆解策略(意图.任务类型)

            // 2b. 查询历史相似任务模板
            历史模板 = 向 ag-ecc-05 查询历史任务模板(意图.任务类型, 意图.实体列表)
            IF 历史模板 != None:
                参考步骤序列 = 历史模板.步骤序列
            ELSE:
                参考步骤序列 = None

            // 2c. 生成步骤序列
            步骤序列 = 基于策略拆解(意图, 拆解策略, 参考步骤序列)

            // 2d. 评估步骤依赖与资源需求
            FOR EACH 步骤 IN 步骤序列:
                步骤.依赖步骤 = 识别前置依赖(步骤, 步骤序列)
                步骤.预估工具类型 = 推断所需工具类型(步骤)

            // 2e. 获取工具能力约束
            SET internal_state = STATE_AWAIT_RES
            工具约束 = 向 ag-ecc-03 查询工具能力约束(步骤序列)
            调整步骤序列适配工具约束(步骤序列, 工具约束)

            // 2f. 构建任务执行计划
            执行计划 = 构建执行计划(计划ID, 意图, 步骤序列)

            // 第3步：安全审查
            向 ag-ecc-04 提交任务执行计划(执行计划)
            安全审查结果 = 等待审查回执()

            IF 安全审查结果.未通过:
                向用户返回任务拒绝通知(安全审查结果.拒绝原因)
                SET internal_state = STATE_WAIT
                CONTINUE

            // 第4步：下发计划
            向 ag-ecc-03 发送步骤资源需求(执行计划.步骤序列)
            活跃任务表[计划ID] = {计划: 执行计划, 状态: "执行中", 当前步骤: 0}

            // 第5步：按优先级插入任务队列
            任务队列.按优先级插入(执行计划)

            SET internal_state = STATE_EXEC
            CONTINUE

        // 第6步：处理步骤执行回执
        IF 收到 ag-ecc-06 步骤执行回执:
            SET internal_state = STATE_EXEC
            回执 = 执行回执
            任务 = 活跃任务表[回执.计划ID]

            IF 回执.执行状态 == "成功":
                任务.当前步骤 += 1
                IF 任务.当前步骤 >= 任务.计划.步骤序列.size:
                    // 全部步骤完成
                    任务.状态 = "已完成"
                    向用户返回任务完成通知
                    从活跃任务表中移除
                ELSE:
                    继续下发下一步骤
            ELSE IF 回执.执行状态 == "失败":
                SET internal_state = STATE_ERR
                恢复计划 = 生成恢复计划(任务.计划, 回执.步骤ID)
                IF 恢复计划 != None:
                    向 ag-ecc-06 发送恢复计划
                ELSE:
                    任务.状态 = "失败"
                    向用户返回任务失败通知

        // 第7步：处理自主目标任务
        IF 收到 ag-ecc-09 自主目标任务:
            自主意图 = 转化为结构化意图(自主目标)
            加入任务队列（优先级低于用户任务）

        // 第8步：周期性状态上报
        IF 距上次状态上报 >= 30秒:
            向 ag-ecc-12 上报规划状态

        SLEEP 10ms


FUNCTION 基于策略拆解(意图, 策略, 历史模板):
    步骤序列 = []
    SWITCH 意图.任务类型:
        CASE "信息查询":
            步骤序列.append({描述: "确定查询源", 工具类型: "SEARCH"})
            步骤序列.append({描述: "执行查询: " + 意图.实体.主题, 工具类型: "API"})
            步骤序列.append({描述: "整理查询结果", 工具类型: "FORMAT"})
        CASE "工具调用":
            步骤序列.append({描述: "准备调用参数", 工具类型: "PARAM"})
            步骤序列.append({描述: "执行工具调用", 工具类型: "API"})
            步骤序列.append({描述: "校验返回结果", 工具类型: "VALIDATE"})
        CASE "内容创作":
            步骤序列.append({描述: "澄清创作需求", 工具类型: "DIALOGUE"})
            步骤序列.append({描述: "规划内容大纲", 工具类型: "REASON"})
            步骤序列.append({描述: "生成内容", 工具类型: "GENERATE"})
            步骤序列.append({描述: "润色与排版", 工具类型: "FORMAT"})
        CASE "对话交互":
            步骤序列.append({描述: "生成回复", 工具类型: "DIALOGUE"})
        DEFAULT:
            步骤序列.append({描述: "执行通用任务", 工具类型: "GENERAL"})

    RETURN 步骤序列
```


## 约束与异常处理

| 场景 | 处理方式 | 恢复条件 |
|------|----------|----------|
| 意图描述的置信度过低（<0.5） | 暂缓规划，向用户发送确认请求 | 用户确认或意图更新 |
| 工具能力约束无法满足某步骤需求 | 尝试替换等效工具，若无等效工具则向用户报告无法执行 | 工具注册中心新增可用工具 |
| 步骤执行失败且无法自动恢复 | 暂停任务，向用户报告失败原因与已完成进度 | 用户指示重试、跳过或取消 |
| 任务队列积压超过阈值（>20个） | 优先处理高优先级任务，低优先级任务通知用户预估等待时间 | 队列消化至正常水平 |
| 历史任务模板缺失（首次遇到该类任务） | 使用通用拆解策略生成计划，标记为"探索模式" | 积累同类任务经验后优化 |
| 紧急熔断 | 暂停所有规划活动，保存当前任务队列状态 | 紧急解除后恢复，继续执行未完成任务 |

## 安全边界

| 规则编号 | 内容 |
|:---:|------|
| T-01 | 所有任务执行计划在下发前必须通过 ag-ecc-04 安全仲裁模块的审查，未通过的计划不得执行 |
| T-02 | 涉及系统配置修改或敏感数据访问的步骤，必须在计划中明确标注"需用户确认" |
| T-03 | 任务队列中的用户显式任务优先级永远高于系统自主生成的任务 |
| T-04 | 任务执行计划中的预估耗时不得作为硬性超时截止时间，仅用于进度展示 |
| T-05 | 本模块仅负责步骤的逻辑拆解，不得直接调用任何工具或访问外部资源 |


## 接口校验用例

| 用例编号 | 前置条件 | 输入 | 预期输出 |
|----------|----------|------|----------|
| TC-E02-01 | `WAITING_INTENT`，收到工具调用意图 | 意图描述（任务类型=工具调用，实体=天气查询） | 生成3步串行计划（参数准备→API调用→结果校验） |
| TC-E02-02 | `WAITING_INTENT`，收到信息查询意图 | 意图描述（任务类型=信息查询，实体=AI架构） | 生成3步计划（确定源→查询→整理） |
| TC-E02-03 | `WAITING_INTENT`，收到对话交互意图 | 意图描述（任务类型=对话交互） | 生成单步计划（生成回复） |
| TC-E02-04 | `EXECUTING`，收到步骤失败回执 | 步骤执行回执（状态=失败，步骤=第2步） | 进入ERROR_RECOVERY，生成恢复计划 |
| TC-E02-05 | `WAITING_INTENT`，收到自主目标任务 | 自主目标（描述=发现新工具可用） | 生成探索任务计划，优先级低于用户任务 |
| TC-E02-06 | `SYSTEM_PAUSED`，收到任何输入 | 任意输入 | 拒绝处理，保持暂停状态 |
