# ag-ecc-01-意图解析模块 接口规格

---

## 基本信息

| 项 | 内容 |
|----|------|
| 模块编号 | ag-ecc-01 |
| 模块名称 | 意图解析模块 |
| 所属分区 | 一、认知大脑核心模块 |
| 核心职责 | 将用户自然语言输入转化为结构化的意图描述，提取核心任务类型、实体参数、约束条件及优先级。为任务规划模块（ag-ecc-02）提供标准化的任务起点，为工具选择模块（ag-ecc-03）提供初步的工具需求线索。支持多轮对话上下文的意图追踪与修正。不参与任务规划或工具执行，仅负责意图的解析与结构化输出 |
| 依赖模块 | ag-ecc-05（记忆查询模块，获取用户历史偏好辅助意图消歧）、ag-ecc-10（社会心智模块，获取用户情绪状态辅助意图理解） |
| 被依赖模块 | ag-ecc-02（任务规划模块，接收结构化意图）、ag-ecc-03（工具选择模块，参考工具需求线索） |


## 内部状态定义

| 状态 | 标识 | 含义 | 触发条件 |
|------|------|------|----------|
| 等待输入 | `WAITING_INPUT` | 无待处理的用户输入 | 系统初始化完成，等待用户消息 |
| 解析中 | `PARSING` | 正在解析用户输入的意图 | 收到用户自然语言输入 |
| 消歧中 | `DISAMBIGUATING` | 意图存在多种可能，正在结合上下文消歧 | 解析结果置信度低于阈值，或存在多个候选意图 |
| 解析完成 | `PARSED` | 意图解析完成，已输出结构化意图 | 解析流程结束 |
| 暂停服务 | `SYSTEM_PAUSED` | 系统紧急熔断 | 收到紧急熔断指令 |


## 输入数据

| 输入项 | 数据类型 | 来源模块 | 触发条件 | 优先级 |
|--------|----------|----------|----------|:---:|
| 用户自然语言输入 | Struct（会话ID + 原始文本 + 输入方式 + 时间戳） | 用户交互层（系统入口） | 用户每次发送消息时 | **高** |
| 用户历史偏好摘要 | Struct（偏好关键词集合 + 高频任务类型 + 常用工具列表） | ag-ecc-05 记忆查询模块 | 解析中按需查询 | 普通 |
| 用户情绪状态数据 | Struct（情绪标签 + 情绪置信度 + 意图倾向） | ag-ecc-10 社会心智模块 | 解析中按需查询 | 普通 |
| 多轮对话上下文 | Struct（会话ID + 最近N轮对话历史 + 已确认的上下文实体） | ag-ecc-07 工作记忆模块 | 每次解析时查询 | **高** |
| 全局调度指令 | Enum（暂停/恢复/熔断） | ag-ecc-12 资源调度模块 | 模式切换或紧急事件时 | **紧急** |


## 输出数据

| 输出项 | 数据类型 | 目标模块 | 输出条件 | 优先级 |
|--------|----------|----------|----------|:---:|
| 结构化意图描述 | Struct（意图ID + 会话ID + 任务类型 + 实体列表 + 约束条件 + 优先级 + 置信度 + 备选意图列表） | ag-ecc-02 任务规划模块 | 解析完成时 | **高** |
| 工具需求线索 | Struct（建议工具类别 + 必需工具特征 + 工具参数预填值） | ag-ecc-03 工具选择模块 | 解析完成时（可选） | 普通 |
| 意图解析状态上报 | Struct（当前状态 + 解析耗时 + 消歧次数 + 最终置信度） | ag-ecc-12 资源调度模块 | 周期性或状态变更时 | 普通 |


## 意图分类体系

### 一、任务类型定义

| 任务类型 | 说明 | 典型触发词 | 所需参数 |
|----------|------|------|------|
| 信息查询 | 获取特定信息或知识 | 什么是、如何、查询、搜索、找 | 查询主题、信息范围、时效要求 |
| 工具调用 | 执行具体操作或调用外部API | 执行、调用、运行、操作 | 工具名称、操作参数、执行条件 |
| 内容创作 | 生成文本、代码、图像等内容 | 写、生成、创作、画、翻译、总结 | 创作主题、风格要求、长度约束 |
| 对话交互 | 日常交流、闲聊、咨询 | 你好、谢谢、怎么样、聊聊 | 交互主题、情感需求 |
| 任务管理 | 对已有任务的操作 | 取消、修改、重试、查看 | 目标任务ID、修改内容 |
| 系统配置 | 调整系统设置或偏好 | 设置、配置、开启、关闭 | 配置项、目标值 |

### 二、意图置信度分级

| 置信度区间 | 级别 | 处理策略 |
|:---:|:---:|------|
| ≥ 0.85 | 高置信度 | 直接输出，无需消歧 |
| 0.60 – 0.84 | 中等置信度 | 输出主意图，附带备选意图列表 |
| < 0.60 | 低置信度 | 触发消歧流程，向用户确认或结合上下文判断 |


## 核心处理逻辑

```
FUNCTION intent_parser_main_loop():
    STATE_WAIT = WAITING_INPUT
    STATE_PARSE = PARSING
    STATE_DISAMBIG = DISAMBIGUATING
    STATE_DONE = PARSED
    STATE_PAUSED = SYSTEM_PAUSED

    SET internal_state = STATE_WAIT
    初始化会话上下文缓存 = {}

    WHILE 系统运行中:
        // 第1步：紧急熔断
        IF 收到紧急熔断指令:
            SET internal_state = STATE_PAUSED
            CONTINUE
        ELSE IF 收到恢复指令 AND internal_state == STATE_PAUSED:
            SET internal_state = STATE_WAIT

        // 第2步：接收用户输入
        IF 收到用户自然语言输入:
            SET internal_state = STATE_PARSE
            原始文本 = 输入.原始文本
            会话ID = 输入.会话ID

            // 第3步：文本预处理
            分词结果 = 分词与词性标注(原始文本)
            命名实体 = 实体识别(原始文本)

            // 第4步：获取对话上下文
            上下文 = 向 ag-ecc-07 查询多轮对话上下文(会话ID)
            最近意图 = 上下文.最近意图
            已确认实体 = 上下文.已确认实体

            // 第5步：意图分类
            候选意图列表 = 基于规则分类(分词结果, 命名实体, 最近意图)

            // 第6步：置信度评估
            FOR EACH 候选意图 IN 候选意图列表:
                候选意图.置信度 = 计算置信度(候选意图, 上下文, 已确认实体)

            // 按置信度排序
            候选意图列表 = 按置信度降序排列(候选意图列表)
            主意图 = 候选意图列表[0]

            // 第7步：消歧判定
            IF 主意图.置信度 < 0.60:
                SET internal_state = STATE_DISAMBIG

                // 7a. 尝试从历史偏好消歧
                历史偏好 = 向 ag-ecc-05 查询用户历史偏好(会话ID)
                IF 历史偏好 != None:
                    主意图 = 基于偏好修正意图(候选意图列表, 历史偏好)

                // 7b. 尝试从情绪状态消歧
                IF 主意图.置信度 < 0.60:
                    情绪数据 = 向 ag-ecc-10 查询用户情绪状态(会话ID)
                    IF 情绪数据 != None:
                        主意图 = 基于情绪修正意图(候选意图列表, 情绪数据)

                // 7c. 仍无法消歧时，生成确认问询
                IF 主意图.置信度 < 0.60:
                    消歧问询 = 生成消歧问询(候选意图列表[:3])
                    向用户发送消歧问询
                    SET internal_state = STATE_WAIT
                    CONTINUE

            // 第8步：输出结构化意图
            SET internal_state = STATE_DONE
            结构化意图 = 构建结构化意图描述(主意图, 候选意图列表[1:3])

            // 第9步：发送至下游模块
            向 ag-ecc-02 发送结构化意图描述
            IF 主意图.任务类型 == "工具调用":
                向 ag-ecc-03 发送工具需求线索

            // 第10步：更新对话上下文
            向 ag-ecc-07 更新对话上下文(会话ID, 结构化意图)

            SET internal_state = STATE_WAIT

        SLEEP 10ms


FUNCTION 基于规则分类(分词结果, 命名实体, 最近意图):
    候选列表 = []

    // 规则1：工具调用类关键词
    IF 分词结果 包含 ["执行", "调用", "运行", "API", "操作"]:
        候选列表.append({任务类型: "工具调用", 置信度: 0.90, 依据: "包含工具调用关键词"})

    // 规则2：信息查询类关键词
    IF 分词结果 包含 ["什么是", "如何", "搜索", "查询", "查找"]:
        候选列表.append({任务类型: "信息查询", 置信度: 0.80, 依据: "包含信息查询关键词"})

    // 规则3：内容创作类关键词
    IF 分词结果 包含 ["写", "生成", "创作", "画", "翻译", "总结"]:
        候选列表.append({任务类型: "内容创作", 置信度: 0.85, 依据: "包含内容创作关键词"})

    // 规则4：对话交互（无明确任务关键词）
    IF 候选列表为空 AND 分词结果 包含对话类词:
        候选列表.append({任务类型: "对话交互", 置信度: 0.70, 依据: "无明确任务关键词，归为对话交互"})

    // 规则5：结合最近意图修正
    IF 最近意图 != None AND 候选列表.size >= 1:
        IF 最近意图.任务类型 匹配 候选列表:
            匹配候选项.置信度 += 0.05
            匹配候选项.依据 += "，与上文意图一致"

    RETURN 候选列表
```


## 约束与异常处理

| 场景 | 处理方式 | 恢复条件 |
|------|----------|----------|
| 用户输入为空或仅包含无意义字符 | 返回默认意图（任务类型=对话交互，置信度=0.3），提示用户重新输入 | 收到有效输入 |
| 意图置信度低于0.60且无法通过上下文消歧 | 生成消歧问询，向用户确认意图 | 用户回复确认 |
| 对话上下文查询超时 | 跳过上下文修正，仅基于当前输入解析 | 查询恢复正常 |
| 情绪状态数据不可用 | 跳过情绪修正步骤，仅使用其他消歧手段 | — |
| 紧急熔断 | 暂停解析，未完成的输入丢弃 | 紧急解除后恢复 |

| MemoryBus | 读 | 用户情绪状态数据 | 只读 | ag-ecc-10 返回 |

## 安全边界

| 规则编号 | 内容 |
|:---:|------|
| P-01 | 意图解析仅基于用户输入文本的语义特征，不得泄露用户的原始输入内容给任何外部系统 |
| P-02 | 消歧过程中获取的用户历史偏好数据仅用于置信度修正，不得作为意图判定的唯一依据 |
| P-03 | 低置信度意图（<0.60）必须触发消歧流程，不得直接输出给下游模块执行 |
| P-04 | 本模块仅输出意图描述，不得直接触发任何工具调用或任务执行 |
| P-05 | 会话上下文数据仅保留最近10轮对话，超出的历史数据自动清除 |


## 接口校验用例

| 用例编号 | 前置条件 | 输入 | 预期输出 |
|----------|----------|------|----------|
| TC-E01-01 | `WAITING_INPUT`，明确工具调用 | 用户输入："帮我调用天气API查询北京天气" | 返回任务类型="工具调用"，置信度≥0.85，含工具需求线索 |
| TC-E01-02 | `WAITING_INPUT`，明确信息查询 | 用户输入："什么是EM-Core架构" | 返回任务类型="信息查询"，置信度≥0.75 |
| TC-E01-03 | `WAITING_INPUT`，模糊输入 | 用户输入："嗯" | 返回任务类型="对话交互"，置信度≤0.50 |
| TC-E01-04 | `PARSING`，低置信度需消歧 | 用户输入："帮我弄一下那个" + 无上下文 | 进入DISAMBIGUATING状态，向用户发送消歧问询 |
| TC-E01-05 | `WAITING_INPUT`，有上下文的多轮对话 | 用户输入："再查一下上海的呢" + 上文有天气查询意图 | 置信度上调0.05，依据含"与上文意图一致" |
| TC-E01-06 | `SYSTEM_PAUSED`，收到用户输入 | 用户输入任意内容 | 拒绝处理，保持暂停状态 |
