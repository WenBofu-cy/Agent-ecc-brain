# ag-ecc-07-工作记忆模块 接口规格

---

## 基本信息

| 项 | 内容 |
|----|------|
| 模块编号 | ag-ecc-07 |
| 模块名称 | 工作记忆模块 |
| 所属分区 | 一、认知大脑核心模块 |
| 核心职责 | 作为 ECC 认知大脑的短期上下文缓存中枢，为当前活跃会话提供临时的数据存储与检索服务。存储推理中间数据、任务进度状态、多轮对话上下文、待确认的决策候选等短期信息。在会话结束时或任务完成后自动清空对应数据。同时负责将高价值的短期认知洞察通过 ag-ecc-05（记忆查询模块）沉淀至 MLNF-Mem 记忆中枢。不参与任何推理或决策，仅提供数据的临时存储、检索与生命周期管理 |
| 依赖模块 | ag-ecc-01（意图解析模块，读写对话上下文）、ag-ecc-02（任务规划模块，读写任务进度状态）、ag-ecc-03（工具选择模块，读写工具评估中间数据）、ag-ecc-05（记忆查询模块，将高价值临时数据沉淀至长期记忆） |
| 被依赖模块 | ag-ecc-01（读取对话上下文）、ag-ecc-02（读写任务进度）、ag-ecc-03（读写候选工具评估数据）、ag-ecc-06（结果评估模块，读取预期结果定义）、ag-ecc-08（元认知模块，读取近期推理链） |


## 内部状态定义

| 状态 | 标识 | 含义 | 触发条件 |
|------|------|------|----------|
| 正常服务 | `NORMAL` | 工作内存正常，接受读写请求 | 系统初始化完成 |
| 容量预警 | `CAPACITY_WARNING` | 某会话的缓存数据量接近上限 | 会话条目数 ≥ 上限的 80% |
| 会话清理中 | `SESSION_CLEANUP` | 正在清理已结束会话的数据 | 会话结束或任务完成 |
| 暂停服务 | `SYSTEM_PAUSED` | 系统紧急熔断 | 收到紧急熔断指令 |


## 输入数据

| 输入项 | 数据类型 | 来源模块 | 触发条件 | 优先级 |
|--------|----------|----------|----------|:---:|
| 上下文写入请求 | Struct（会话ID + 数据类型：对话上下文/任务进度/工具评估/推理链 + 数据负载 + 保留策略） | ag-ecc-01/02/03/06/08 等 | 各模块需要暂存中间数据时 | **高** |
| 上下文查询请求 | Struct（会话ID + 查询范围 + 最大返回条数 + 时间窗口） | ag-ecc-01/02/03/06/08 等 | 各模块需要回溯上下文时 | **高** |
| 会话结束通知 | Struct（会话ID + 结束原因 + 是否强制清理） | 系统会话管理器或 ag-ecc-02 | 用户会话结束时 | **高** |
| 任务完成通知 | Struct（计划ID + 完成状态 + 关联会话ID） | ag-ecc-02 任务规划模块 | 任务执行完成时 | 普通 |
| 长效记忆沉淀指令 | Struct（待沉淀条目ID列表 + 沉淀类型 + 优先级） | ag-ecc-02 或 ag-ecc-08 | 判定数据具有长期学习价值时 | 普通 |
| 全局调度指令 | Enum（暂停/恢复/熔断） | ag-ecc-12 资源调度模块 | 模式切换或紧急事件时 | **紧急** |


## 输出数据

| 输出项 | 数据类型 | 目标模块 | 输出条件 | 优先级 |
|--------|----------|----------|----------|:---:|
| 查询结果 | Struct（会话ID + 匹配的上下文条目列表 + 条目时间戳 + 来源模块） | 请求来源模块 | 查询完成时 | **高** |
| 写入确认 | Struct（条目ID + 写入状态 + 当前会话使用率） | 请求来源模块 | 写入完成时 | 普通 |
| 长效沉淀条目列表 | Struct（条目列表 + 各条目内容 + 来源模块 + 原始时间戳） | ag-ecc-05 记忆查询模块 | 触发沉淀指令时 | 普通 |
| 工作内存状态上报 | Struct（当前状态 + 活跃会话数 + 总条目数 + 总使用率 + 高价值沉淀次数） | ag-ecc-12 资源调度模块 | 周期性（每60秒）或状态变更时 | 普通 |


## 数据分类与保留策略

| 数据类型 | 存储结构 | 单条最大大小 | 每会话最大条目 | 保留策略 |
|----------|------|:---:|:---:|------|
| 对话上下文 | 消息历史列表（轮次+发言者+内容摘要+时间戳） | 2KB | 50条（最近10轮对话） | 会话结束立即清除 |
| 任务进度状态 | 步骤状态列表（步骤ID+完成状态+输出摘要） | 1KB | 100条 | 任务完成或失败后保留至会话结束 |
| 工具评估中间数据 | 候选工具评分表（工具名+各维度评分+综合分） | 3KB | 30条 | 步骤完成立即清除 |
| 推理链 | 推理步骤序列（步骤描述+置信度+依赖关系） | 1KB | 20条 | 任务完成立即清除 |
| 待确认决策候选 | 候选方案列表（方案ID+方案描述+风险评估） | 1KB | 10条 | 决策确认后立即清除 |


## 核心处理逻辑

```
FUNCTION working_memory_main_loop():
    STATE_NORMAL = NORMAL
    STATE_WARN = CAPACITY_WARNING
    STATE_CLEANUP = SESSION_CLEANUP
    STATE_PAUSED = SYSTEM_PAUSED

    SET internal_state = STATE_NORMAL
    初始化会话存储表 = {}  // 会话ID → 数据分类 → 条目列表
    初始化会话元数据表 = {}  // 会话ID → {创建时间, 最后活跃时间, 任务列表}

    WHILE 系统运行中:
        // 第1步：紧急熔断
        IF 收到紧急熔断指令:
            SET internal_state = STATE_PAUSED
            CONTINUE
        ELSE IF 收到恢复指令 AND internal_state == STATE_PAUSED:
            SET internal_state = STATE_NORMAL

        // 第2步：处理写入请求
        IF 收到上下文写入请求:
            会话ID = 请求.会话ID
            数据类型 = 请求.数据类型
            数据负载 = 请求.数据负载

            // 2a. 初始化会话存储
            IF 会话ID NOT IN 会话存储表:
                会话存储表[会话ID] = {}
                会话元数据表[会话ID] = {创建时间: NOW(), 最后活跃时间: NOW(), 任务列表: []}

            // 2b. 容量检查
            IF 数据类型 NOT IN 会话存储表[会话ID]:
                会话存储表[会话ID][数据类型] = []

            当前会话条目数 = SUM(各类型条目数)
            IF 当前会话条目数 >= 每会话最大总条目 * 0.8:
                SET internal_state = STATE_WARN
                触发温和清理(会话ID)
                SET internal_state = STATE_NORMAL

            // 2c. 追加条目
            条目ID = 生成条目ID()
            条目 = {条目ID, 数据负载, 时间戳: NOW(), 来源模块: 请求.来源模块}
            会话存储表[会话ID][数据类型].append(条目)

            // 2d. 更新元数据
            会话元数据表[会话ID].最后活跃时间 = NOW()

            返回写入确认(条目ID, 当前会话使用率)

        // 第3步：处理查询请求
        IF 收到上下文查询请求:
            IF 请求.会话ID NOT IN 会话存储表:
                返回空结果
                CONTINUE

            匹配条目 = 按条件检索(会话存储表[请求.会话ID], 请求.查询范围)
            匹配条目 = 按时间戳排序(匹配条目)
            匹配条目 = 截断到最大返回条数(匹配条目, 请求.最大返回条数)

            返回查询结果(匹配条目)

        // 第4步：处理会话结束
        IF 收到会话结束通知:
            SET internal_state = STATE_CLEANUP
            会话ID = 通知.会话ID

            // 4a. 检查是否有待沉淀的高价值数据
            高价值条目 = 筛选高价值数据(会话存储表[会话ID])
            IF 高价值条目 非空:
                向 ag-ecc-05 发送长效沉淀条目列表(高价值条目)

            // 4b. 清除会话数据
            DEL 会话存储表[会话ID]
            DEL 会话元数据表[会话ID]

            SET internal_state = STATE_NORMAL

        // 第5步：处理任务完成
        IF 收到任务完成通知:
            // 清除该任务对应的工具评估中间数据、推理链、待确认决策
            清除任务相关数据(通知.计划ID)

        // 第6步：周期性状态上报
        IF 距上次状态上报 >= 60秒:
            向 ag-ecc-12 上报工作内存状态

        SLEEP 10ms
```


## 约束与异常处理

| 场景 | 处理方式 | 恢复条件 |
|------|----------|----------|
| 某会话条目数达到上限 | 温和清理：按时间戳淘汰最旧的条目，优先保留对话上下文和任务进度 | 条目数降至上限以下 |
| 会话超过 30 分钟无活跃 | 自动标记为"可能已结束"，触发长效沉淀后清理 | 用户重新活跃时恢复 |
| 查询请求指定了不存在的会话ID | 返回空结果，不报错 | — |
| 写入请求的数据负载超过单条大小上限 | 拒绝写入，返回"数据超限"错误 | 请求方缩减数据大小后重试 |
| 紧急熔断 | 暂停所有操作，内存数据不丢失（仅内存存储） | 紧急解除后恢复 |


## 总线契约

| 总线 | 操作 | 数据内容 | 权限 | 说明 |
|------|------|----------|------|------|
| MemoryBus | 读 | 上下文写入/查询请求 | 只读 | ag-ecc-01/02/03/06/08 发送 |
| MemoryBus | 读 | 会话结束/任务完成通知 | 只读 | 会话管理器或 ag-ecc-02 发送 |
| MemoryBus | 写 | 查询结果 / 写入确认 | 专属写入 | 向请求来源返回 |
| MemoryBus | 写 | 长效沉淀条目列表 | 专属写入 | 向 ag-ecc-05 发送 |
| MemoryBus | 写 | 工作内存状态上报 | 周期性写入 | 向 ag-ecc-12 发送 |


## 安全边界

| 规则编号 | 内容 |
|:---:|------|
| W-01 | 工作内存数据仅存于内存，进程重启后自动丢弃，不进行任何持久化存储 |
| W-02 | 不同会话之间的工作内存数据严格隔离，禁止跨会话查询或数据混合 |
| W-03 | 会话结束时，对话上下文和用户原始输入数据必须立即清除，不得残留 |
| W-04 | 长效沉淀至记忆中枢的数据必须经过去个性化处理，不得包含用户个人身份信息 |
| W-05 | 工作内存中不得缓存任何安全令牌、密钥或用户密码等敏感凭证信息 |
| W-06 | 单个会话的总条目数不得超过硬上限，防止内存溢出 |
