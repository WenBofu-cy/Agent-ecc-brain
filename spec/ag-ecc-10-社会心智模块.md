# ag-ecc-10-社会心智模块 接口规格

---

## 基本信息

| 项 | 内容 |
|----|------|
| 模块编号 | ag-ecc-10 |
| 模块名称 | 社会心智模块 |
| 所属分区 | 一、认知大脑核心模块 |
| 核心职责 | 作为 ECC 认知大脑的人机交互专属心智单元，负责感知用户实时情绪状态、推断行为意图、理解社交上下文，并据此适配系统的交互风格（语气、节奏、表达方式）。通过查询用户长期交互历史与个人偏好，实现千人千面的个性化交互体验。在检测到用户负面情绪或交互困境时，调整交互策略并向相关模块发出适配建议。不参与任务规划或工具执行，仅负责社会认知与交互风格的智能适配 |
| 依赖模块 | ag-ecc-05（记忆查询模块，获取用户交互历史与偏好）、ag-ecc-01（意图解析模块，可参考社会心智数据辅助意图消歧）、ag-mem-46（用户情绪意图感知库，获取实时情绪与意图推断结果）、ag-mem-11（个性化建议生成单元，获取用户偏好背景） |
| 被依赖模块 | ag-ecc-01（意图解析模块，接收情绪与意图预判数据）、ag-ecc-04（安全仲裁模块，可参考社交风险等级）、ag-mem-11（个性化建议生成单元，接收交互风格适配建议）、ag-ecc-12（资源调度模块，接收社交异常告警） |


## 内部状态定义

| 状态 | 标识 | 含义 | 触发条件 |
|------|------|------|----------|
| 正常监控 | `NORMAL_MONITOR` | 用户情绪平稳，交互正常 | 系统初始化完成，用户情绪标签为平静或满意 |
| 情绪关注 | `EMOTION_ATTENTION` | 检测到用户情绪波动，正在分析原因 | 用户情绪标签变为困惑、焦虑或失望 |
| 冲突化解 | `CONFLICT_RESOLUTION` | 检测到用户愤怒或强烈不满，启动安抚策略 | 用户情绪标签变为愤怒，或连续表达不满 |
| 交互优化 | `INTERACTION_OPTIMIZING` | 正在基于用户偏好调整交互策略 | 用户画像数据可用，或检测到可优化的交互模式 |
| 暂停服务 | `SYSTEM_PAUSED` | 系统紧急熔断 | 收到紧急熔断指令 |

## 交互风格适配规则

### 一、情绪-风格映射

| 用户情绪 | 建议语气 | 建议交互节奏 | 主动推送倾向 | 特殊处理 |
|----------|------|:---:|:---:|------|
| 平静 | 正常 | 正常响应 | 正常 | 无特殊处理 |
| 满意 | 简洁友好 | 快速响应 | 可适度推送 | 可增加主动建议 |
| 困惑 | 耐心详细 | 缓慢引导 | 暂停推送 | 主动提供帮助选项 |
| 焦虑 | 安抚简洁 | 快速响应 | 暂停推送 | 优先处理当前问题 |
| 愤怒 | 冷静中立 | 缓慢引导 | 停止推送 | 启动冲突化解策略 |
| 失望 | 诚恳温和 | 正常响应 | 暂停推送 | 主动承认不足并提供补偿方案 |

### 二、交互节奏参数

| 节奏类型 | 回复速度 | 确认频率 | 信息密度 | 适用场景 |
|----------|:---:|:---:|:---:|------|
| 快速响应 | 即时回复 | 减少冗余确认 | 简洁直接 | 用户熟悉系统、紧急任务 |
| 正常响应 | 标准节奏 | 重要步骤确认 | 适中 | 日常交互 |
| 缓慢引导 | 逐步回复 | 每步确认 | 详细解释 | 新用户、复杂任务、用户困惑时 |


## 核心处理逻辑

```
FUNCTION social_mind_main_loop():
    STATE_NORMAL = NORMAL_MONITOR
    STATE_EMOTION = EMOTION_ATTENTION
    STATE_CONFLICT = CONFLICT_RESOLUTION
    STATE_OPTIMIZE = INTERACTION_OPTIMIZING
    STATE_PAUSED = SYSTEM_PAUSED

    SET internal_state = STATE_NORMAL
    初始化用户情绪历史 = {}  // 会话ID → 最近情绪序列
    初始化交互策略表 = {}    // 会话ID → 当前交互策略

    WHILE 系统运行中:
        // 第1步：紧急熔断
        IF 收到紧急熔断指令:
            SET internal_state = STATE_PAUSED
            CONTINUE
        ELSE IF 收到恢复指令 AND internal_state == STATE_PAUSED:
            SET internal_state = STATE_NORMAL

        // 第2步：接收用户情绪意图感知结果
        IF 收到 ag-mem-46 情绪意图感知结果:
            会话ID = 感知结果.会话ID
            情绪标签 = 感知结果.情绪标签
            意图倾向 = 感知结果.意图倾向

            // 更新情绪历史
            IF 会话ID NOT IN 用户情绪历史:
                用户情绪历史[会话ID] = []
            用户情绪历史[会话ID].append({情绪: 情绪标签, 时间: NOW()})
            // 仅保留最近10条
            用户情绪历史[会话ID] = 用户情绪历史[会话ID][-10:]

            // 状态判定
            IF 情绪标签 == "愤怒":
                SET internal_state = STATE_CONFLICT
                启动冲突化解(会话ID)
            ELSE IF 情绪标签 IN ("困惑", "焦虑", "失望"):
                SET internal_state = STATE_EMOTION

            // 生成交互风格适配建议
            当前策略 = 交互策略表.GET(会话ID, 默认策略)
            适配建议 = 根据情绪调整策略(情绪标签, 当前策略, 意图倾向)
            交互策略表[会话ID] = 适配建议

            // 发送适配建议
            向 ag-ecc-01 发送交互风格适配建议(适配建议)
            向 ag-mem-11 发送交互风格适配建议(适配建议)

            // 发送情绪与意图预判数据
            向 ag-ecc-01 发送情绪与意图预判数据(情绪标签, 意图倾向)

            // 检查社交风险
            IF 情绪标签 == "愤怒" OR 检测到连续负面情绪():
                向 ag-ecc-04 发送社交风险告警(会话ID, 情绪标签)

            IF internal_state IN (STATE_EMOTION, STATE_CONFLICT):
                SET internal_state = STATE_NORMAL

        // 第3步：会话开始时加载用户交互历史
        IF 检测到新会话:
            交互历史 = 向 ag-ecc-05 查询用户交互历史摘要(用户ID)
            IF 交互历史 != None:
                偏好背景 = 向 ag-mem-11 查询用户偏好背景(用户ID)
                SET internal_state = STATE_OPTIMIZE
                初始策略 = 基于历史构建初始策略(交互历史, 偏好背景)
                交互策略表[会话ID] = 初始策略
                SET internal_state = STATE_NORMAL

        // 第4步：处理社交风险评估请求
        IF 收到社交风险评估请求:
            评估结果 = 评估社交风险(请求.待评估内容)
            向 请求来源 返回评估结果

        // 第5步：周期性状态上报
        IF 距上次状态上报 >= 60秒:
            向 ag-ecc-12 上报社交心智状态

        SLEEP 10ms


FUNCTION 根据情绪调整策略(情绪标签, 当前策略, 意图倾向):
    映射 = 情绪-风格映射表[情绪标签]
    新策略 = 当前策略.clone()
    新策略.语气 = 映射.建议语气
    新策略.交互节奏 = 映射.建议交互节奏
    新策略.主动推送 = 映射.主动推送倾向
    RETURN 新策略


FUNCTION 启动冲突化解(会话ID):
    // 1. 暂停主动推送
    // 2. 采用冷静中立语气
    // 3. 优先处理当前问题
    // 4. 记录冲突事件日志
    记录社会事件日志("冲突化解启动", 会话ID)
```


## 约束与异常处理

| 场景 | 处理方式 | 恢复条件 |
|------|----------|----------|
| 情绪意图感知库不可用 | 使用最近一次有效数据，标记"情绪数据过时"，采用保守交互策略 | 感知库恢复正常 |
| 用户交互历史为空（新用户） | 使用默认交互策略（正常语气、标准节奏），标记"新用户" | 积累交互数据后优化 |
| 情绪频繁切换（短时间内变化超过3次） | 锁定当前策略，抑制频繁调整，延迟30秒后再重新评估 | 情绪稳定持续30秒 |
| 社交风险告警连续触发 | 升级风险等级，建议安全仲裁模块进行人工审核 | 风险解除 |
| 紧急熔断 | 暂停所有社交评估，保持当前交互策略不变 | 紧急解除后恢复 |

## 安全边界

| 规则编号 | 内容 |
|:---:|------|
| S-01 | 情绪意图感知数据仅用于交互风格适配，不得作为安全决策或任务执行的唯一依据 |
| S-02 | 用户交互历史与偏好数据仅在本地处理，不得上传云端或共享给第三方模块 |
| S-03 | 冲突化解期间，系统不得主动推送任何营销或推广内容 |
| S-04 | 社交风险评估仅作为辅助参考，最终安全决策权归 ag-ecc-04 安全仲裁模块 |
| S-05 | 本模块不得存储用户的原始对话内容，仅保留脱敏后的交互特征摘要 |


## 接口校验用例

| 用例编号 | 前置条件 | 输入 | 预期输出 |
|----------|----------|------|----------|
| TC-E10-01 | `NORMAL_MONITOR`，用户情绪=满意 | 情绪感知结果（情绪=满意，意图=任务执行） | 生成适配建议（简洁友好+快速响应），发送至ag-ecc-01 |
| TC-E10-02 | `NORMAL_MONITOR`，用户情绪=困惑 | 情绪感知结果（情绪=困惑，意图=寻求帮助） | 切换至EMOTION_ATTENTION，适配建议（耐心详细+缓慢引导+主动提供帮助） |
| TC-E10-03 | `NORMAL_MONITOR`，用户情绪=愤怒 | 情绪感知结果（情绪=愤怒，意图=表达不满） | 切换至CONFLICT_RESOLUTION，启动冲突化解，发送社交风险告警 |
| TC-E10-04 | `NORMAL_MONITOR`，新会话启动 | 检测到新会话，有交互历史 | 加载历史→构建初始策略→进入INTERACTION_OPTIMIZING |
| TC-E10-05 | `EMOTION_ATTENTION`，情绪频繁切换 | 短时间内第4次情绪变化 | 锁定当前策略，延迟30秒再评估 |
| TC-E10-06 | `SYSTEM_PAUSED`，收到情绪感知结果 | 情绪感知结果 | 拒绝处理，保持暂停状态 |
