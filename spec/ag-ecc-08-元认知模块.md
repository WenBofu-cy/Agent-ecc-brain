# ag-ecc-08-元认知模块 接口规格

---

## 基本信息

| 项 | 内容 |
|----|------|
| 模块编号 | ag-ecc-08 |
| 模块名称 | 元认知模块 |
| 所属分区 | 一、认知大脑核心模块 |
| 核心职责 | 作为 ECC 认知大脑的自我觉察与自省中枢，负责对系统自身的运行状态、认知能力、决策质量进行实时监控与评估。通过分析任务执行的成功率、响应耗时、异常模式与能力边界，输出系统能力评估报告。在检测到认知偏差、能力缺口或异常模式时，触发自我反思与学习需求，生成能力提升建议。当遇到自身无法解决的认知困境时，主动向用户或大模型发起求助请求。不参与具体任务的推理或执行，仅负责"对认知过程本身的认知" |
| 依赖模块 | ag-ecc-05（记忆查询模块，查询历史失败经验）、ag-ecc-06（结果评估模块，获取任务执行质量数据）、ag-ecc-07（工作记忆模块，读取近期推理链与决策序列）、ag-ecc-12（资源调度模块，发出求助请求与接收大模型调用配额） |
| 被依赖模块 | ag-ecc-02（任务规划模块，反馈能力约束建议）、ag-ecc-09（内生动机模块，输出能力缺口以触发自主学习）、ag-ecc-12（接收求助请求、能力评估报告） |


## 内部状态定义

| 状态 | 标识 | 含义 | 触发条件 |
|------|------|------|----------|
| 正常监控 | `NORMAL_MONITOR` | 系统运行平稳，无异常信号，定期进行能力评估 | 系统初始化完成，各项指标正常 |
| 异常关注 | `ANOMALY_ATTENTION` | 检测到单一异常信号，正在深入分析根因 | 某项指标偏离正常范围（成功率下降、耗时异常等） |
| 能力评估中 | `CAPABILITY_ASSESSING` | 正在系统性评估当前系统各项能力边界 | 触发定期能力评估或收到评估请求 |
| 短板识别 | `GAP_IDENTIFYING` | 已识别出能力短板，正在分析根因与学习需求 | 能力评估发现某维度低于阈值 |
| 主动求助 | `HELP_SEEKING` | 遇到自身无法解决的认知困境，正在发起求助 | 置信度低于求助阈值且无法通过内部机制恢复 |
| 暂停服务 | `SYSTEM_PAUSED` | 系统紧急熔断 | 收到紧急熔断指令 |

## 能力评估维度

### 一、核心能力维度

| 能力维度 | 评估指标 | 数据来源 | 评估频率 | 健康阈值 |
|----------|------|------|:---:|:---:|
| 意图理解能力 | 意图解析准确率 + 消歧成功率 | ag-ecc-01 状态统计 | 每日 | ≥ 85% |
| 任务规划能力 | 任务完成率 + 步骤拆解合理性 | ag-ecc-02 执行回执 | 每日 | ≥ 80% |
| 工具使用能力 | 工具选择准确率 + 调用成功率 | ag-ecc-03/06 统计 | 每日 | ≥ 90% |
| 安全合规能力 | 安全审查拦截率 + 误拦截率 | ag-ecc-04 安全日志 | 每日 | 拦截率 < 5%，误拦率 < 1% |
| 记忆检索能力 | 记忆查询命中率 + 查询响应耗时 | ag-ecc-05 查询统计 | 每日 | 命中率 ≥ 70% |
| 整体决策质量 | 用户满意度（点赞/点踩比例） | 用户反馈系统 | 每周 | 满意度 ≥ 80% |

### 二、认知偏差类型

| 偏差类型 | 判定条件 | 严重度 |
|----------|------|:---:|
| 过度自信 | 连续多次高置信度预测但实际成功率偏低 | 中 |
| 过度保守 | 频繁触发消歧问询且用户多次选择"直接执行" | 低 |
| 经验依赖 | 过于依赖历史模板而忽略新工具或新方案 | 中 |
| 能力误判 | 反复尝试已被标记为"失败不可恢复"的同类任务 | 高 |


## 核心处理逻辑

```
FUNCTION metacognition_main_loop():
    STATE_NORMAL = NORMAL_MONITOR
    STATE_ANOMALY = ANOMALY_ATTENTION
    STATE_ASSESS = CAPABILITY_ASSESSING
    STATE_GAP = GAP_IDENTIFYING
    STATE_HELP = HELP_SEEKING
    STATE_PAUSED = SYSTEM_PAUSED

    SET internal_state = STATE_NORMAL
    初始化能力评分表 = {各能力维度: 1.0}
    初始化异常计数器 = 0
    初始化求助冷却计时器 = 0

    WHILE 系统运行中:
        // 第1步：紧急熔断
        IF 收到紧急熔断指令:
            SET internal_state = STATE_PAUSED
            CONTINUE
        ELSE IF 收到恢复指令 AND internal_state == STATE_PAUSED:
            SET internal_state = STATE_NORMAL

        // 第2步：周期性能力评估
        IF 收到能力评估触发信号 OR 距上次评估 >= 24小时:
            SET internal_state = STATE_ASSESS

            // 2a. 收集各维度运行指标
            全链路指标 = 汇总各模块运行指标()
            失败经验 = 向 ag-ecc-05 查询近期失败经验()

            // 2b. 计算各维度评分
            FOR EACH 能力维度 IN 能力维度表:
                评分 = 计算维度评分(能力维度, 全链路指标, 失败经验)
                能力评分表[能力维度] = 评分

            // 2c. 识别短板
            短板列表 = []
            FOR EACH 能力维度, 评分 IN 能力评分表:
                IF 评分 < 健康阈值:
                    短板列表.append({维度: 能力维度, 评分: 评分, 差距: 健康阈值 - 评分})

            // 2d. 生成能力评估报告
            向 ag-ecc-12 发送系统能力评估报告(能力评分表, 短板列表)

            // 2e. 短板处理
            IF 短板列表 非空:
                SET internal_state = STATE_GAP
                FOR EACH 短板 IN 短板列表:
                    学习需求 = 生成自主学习需求(短板)
                    向 ag-ecc-09 发送自主学习需求(学习需求)
                SET internal_state = STATE_NORMAL

        // 第3步：异常监控
        近期指标 = 获取近期指标快照()
        IF 检测到异常波动(近期指标):
            SET internal_state = STATE_ANOMALY
            异常计数器++
            偏差类型 = 判定认知偏差类型(近期指标)

            IF 偏差类型 != None:
                向 ag-ecc-09 发送认知偏差告警(偏差类型)
                向 ag-ecc-12 发送认知偏差告警(偏差类型)

            // 如果连续异常超过阈值，触发主动求助
            IF 异常计数器 >= 3:
                IF 求助冷却计时器已过期（> 1小时）:
                    SET internal_state = STATE_HELP
                    求助请求 = 构建求助请求(偏差类型, 近期指标)
                    向 ag-ecc-12 发送求助请求(求助请求)
                    求助冷却计时器 = NOW()
                ELSE:
                    // 冷却中，仅记录日志
                    记录元认知运行日志("求助冷却中", 偏差类型)

            SET internal_state = STATE_NORMAL

        // 第4步：处理用户求助响应
        IF 收到用户求助响应:
            IF 用户回复.是否采纳建议:
                根据用户建议调整能力评估参数
            记录元认知运行日志("用户求助响应", 用户回复)

        // 第5步：处理大模型返回结果
        IF 收到大模型返回结果:
            分析建议 = 解析大模型建议(大模型返回结果)
            向 ag-ecc-09 发送自主学习需求(分析建议)
            记录元认知运行日志("大模型辅助", 分析建议)

        // 第6步：周期性状态上报
        IF 距上次状态上报 >= 120秒:
            向 ag-ecc-12 上报元认知状态

        SLEEP 100ms
```


## 约束与异常处理

| 场景 | 处理方式 | 恢复条件 |
|------|----------|----------|
| 全链路指标获取失败（某模块不可用） | 使用最近一次有效数据，标记"数据可能不完整" | 对应模块恢复正常 |
| 能力评估期间系统负载过高 | 评估降级为仅统计关键维度（意图理解+任务完成率），跳过详细分析 | 系统负载回落 |
| 主动求助无响应（用户不在线且大模型不可用） | 记录求助事件，进入求助冷却期（1小时），期间抑制重复求助 | 冷却期结束或用户上线 |
| 连续检测到同一偏差类型但未见改善 | 升级偏差严重度，从"建议"升级为"强制告警" | 偏差指标改善 |
| 紧急熔断 | 暂停监控与评估，保持当前状态数据 | 紧急解除后恢复 |

## 安全边界

| 规则编号 | 内容 |
|:---:|------|
| M-01 | 本模块仅做监控与建议输出，不直接干预任何业务模块的运行或决策 |
| M-02 | 系统能力评估报告仅保留最近90天的趋势数据，超期自动清除 |
| M-03 | 主动求助时，发送给大模型的困境描述必须经过去个性化处理，不得包含用户原始输入 |
| M-04 | 求助冷却机制确保系统不会在短时间内频繁向用户或大模型发起重复求助 |
| M-05 | 元认知运行日志记录的异常模式不得包含任何可关联到特定用户的身份信息 |
