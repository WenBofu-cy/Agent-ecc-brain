# ag-ecc-03-工具选择模块 接口规格

---

## 基本信息

| 项 | 内容 |
|----|------|
| 模块编号 | ag-ecc-03 |
| 模块名称 | 工具选择模块 |
| 所属分区 | 一、认知大脑核心模块 |
| 核心职责 | 接收 ag-ecc-02（任务规划模块）下发的步骤资源需求，在工具注册中心（ag-mcc-04）中检索匹配的工具清单，综合评估各工具的功能匹配度、调用成本、历史成功率与当前可用性，为每个步骤输出最优工具选择方案（含备选方案）。为 ag-ecc-02 提供工具能力约束信息以辅助任务规划。不参与工具的实际调用执行，仅负责工具的选择与推荐 |
| 依赖模块 | ag-ecc-02（任务规划模块，接收步骤资源需求）、ag-ecc-05（记忆查询模块，查询历史工具使用经验）、ag-mcc-04（工具注册中心，查询可用工具目录与参数约束）、ag-mcc-05（工具参数校验器，预校验参数可行性） |
| 被依赖模块 | ag-ecc-02（返回工具能力约束与选择方案）、ag-ecc-04（安全仲裁模块，提交工具方案供安全审查）、ag-ecc-12（资源调度模块，上报工具调用配额消耗预估） |


## 内部状态定义

| 状态 | 标识 | 含义 | 触发条件 |
|------|------|------|----------|
| 等待需求 | `WAITING_REQUIREMENT` | 无待处理需求，等待 ag-ecc-02 下发 | 系统初始化完成，无待处理任务 |
| 检索匹配中 | `MATCHING` | 正在向工具注册中心发起查询并执行匹配 | 收到步骤资源需求 |
| 评估筛选中 | `EVALUATING` | 正在对候选工具进行多维评估与排序 | 检索完成，存在多个候选工具 |
| 选择完成 | `SELECTED` | 工具选择完成，已输出方案 | 评估完成，选定最优方案 |
| 无可用工具 | `NO_TOOL_AVAILABLE` | 无任何工具能满足当前步骤需求 | 检索结果为空 |
| 暂停服务 | `SYSTEM_PAUSED` | 系统紧急熔断 | 收到紧急熔断指令 |


## 输入数据

| 输入项 | 数据类型 | 来源模块 | 触发条件 | 优先级 |
|--------|----------|----------|----------|:---:|
| 步骤资源需求 | Struct（步骤ID + 计划ID + 所需工具类别 + 参数约束 + 期望完成时间 + 是否允许降级） | ag-ecc-02 任务规划模块 | 任务规划完成时 | **高** |
| 工具目录与约束 | Struct（工具名称 + 功能描述 + 参数模板 + 调用成本 + 当前可用状态 + 最大并发数） | ag-mcc-04 工具注册中心（通过 MemoryBus 查询） | 每次工具选择时查询 | **高** |
| 历史工具使用经验 | Struct（工具名称 + 同类任务成功率 + 平均响应耗时 + 常见失败原因） | ag-ecc-05 记忆查询模块 | 评估候选工具时按需查询 | 普通 |
| 参数预校验结果 | Struct（步骤ID + 工具名称 + 参数是否合法 + 缺失参数列表 + 建议补充方式） | ag-mcc-05 工具参数校验器 | 候选工具初筛后 | 普通 |
| 全局调度指令 | Enum（暂停/恢复/熔断） | ag-ecc-12 资源调度模块 | 模式切换或紧急事件时 | **紧急** |


## 输出数据

| 输出项 | 数据类型 | 目标模块 | 输出条件 | 优先级 |
|--------|----------|----------|----------|:---:|
| 工具选择方案 | Struct（步骤ID + 主选工具 + 备选工具列表 + 各工具匹配度评分 + 预估调用成本 + 预估成功率 + 参数建议值） | ag-ecc-02（返回方案）、ag-ecc-04（安全审查） | 选择完成时 | **高** |
| 工具能力约束 | Struct（可用工具清单 + 各工具参数约束 + 当前负载 + 不可用的工具类别） | ag-ecc-02 任务规划模块 | 规划阶段查询时 | **高** |
| 无工具通知 | Struct（步骤ID + 缺失的工具类别 + 建议替代方案 + 是否可降级） | ag-ecc-02 任务规划模块 | 检索结果为空时 | **高** |
| 选择状态上报 | Struct（当前状态 + 本次选择耗时 + 候选工具数 + 最终匹配度） | ag-ecc-12 资源调度模块 | 每次选择完成或异常时 | 普通 |


## 工具评估维度

### 一、多维评估矩阵

| 评估维度 | 权重 | 评分方式 | 数据来源 |
|----------|:---:|------|------|
| 功能匹配度 | 0.40 | 所需功能与工具能力的语义匹配程度 | ag-mcc-04 工具描述 |
| 历史成功率 | 0.25 | 同类任务中该工具的成功执行比例 | ag-ecc-05 历史经验 |
| 调用成本 | 0.20 | API调用费用、计算资源消耗、预估耗时 | ag-mcc-04 成本标签 |
| 当前可用性 | 0.15 | 工具是否在线、并发配额是否充足 | ag-mcc-04 实时状态 |

### 二、选择策略

| 场景 | 策略 |
|------|------|
| 存在历史成功率 ≥ 90% 的工具 | 直接选择，跳过评估流程 |
| 功能匹配度最高的工具不可用 | 选择功能匹配度次高的可用工具 |
| 所有候选工具成功率均 < 50% | 标记为高风险，附带警告输出 |
| 用户显式指定了工具 | 跳过匹配，直接使用用户指定工具 |
| 无任何匹配工具 | 返回无工具通知，建议任务降级或等待 |


## 核心处理逻辑

```
FUNCTION tool_selector_main_loop():
    STATE_WAIT = WAITING_REQUIREMENT
    STATE_MATCH = MATCHING
    STATE_EVAL = EVALUATING
    STATE_DONE = SELECTED
    STATE_NO_TOOL = NO_TOOL_AVAILABLE
    STATE_PAUSED = SYSTEM_PAUSED

    SET internal_state = STATE_WAIT

    WHILE 系统运行中:
        // 第1步：紧急熔断
        IF 收到紧急熔断指令:
            SET internal_state = STATE_PAUSED
            CONTINUE
        ELSE IF 收到恢复指令 AND internal_state == STATE_PAUSED:
            SET internal_state = STATE_WAIT

        // 第2步：接收步骤资源需求
        IF 收到 ag-ecc-02 步骤资源需求:
            SET internal_state = STATE_MATCH
            需求 = 步骤资源需求
            步骤ID = 需求.步骤ID
            所需类别 = 需求.所需工具类别

            // 2a. 用户指定工具优先
            IF 需求.用户指定工具 != None:
                方案 = 构建用户指定方案(需求.用户指定工具)
                向 ag-ecc-02 返回工具选择方案(方案)
                向 ag-ecc-04 提交工具选择方案(方案)
                SET internal_state = STATE_DONE
                CONTINUE

            // 2b. 向工具注册中心查询
            工具目录 = 向 ag-mcc-04 查询工具目录(所需类别)
            IF 工具目录为空:
                SET internal_state = STATE_NO_TOOL
                向 ag-ecc-02 发送无工具通知(步骤ID, 所需类别, "无可用的匹配工具")
                SET internal_state = STATE_WAIT
                CONTINUE

            // 2c. 初筛：功能匹配度
            候选列表 = 按功能匹配度筛选(工具目录, 需求)
            IF 候选列表为空:
                SET internal_state = STATE_NO_TOOL
                向 ag-ecc-02 发送无工具通知(步骤ID, 所需类别, "无功能匹配的工具")
                SET internal_state = STATE_WAIT
                CONTINUE

            // 第3步：多维评估
            SET internal_state = STATE_EVAL

            FOR EACH 候选工具 IN 候选列表:
                // 3a. 查询历史成功率
                历史经验 = 向 ag-ecc-05 查询工具历史经验(候选工具.名称, 需求.任务类型)
                IF 历史经验 != None:
                    候选工具.历史成功率 = 历史经验.成功率

                // 3b. 参数预校验
                预校验结果 = 向 ag-mcc-05 请求参数预校验(候选工具, 需求.参数约束)
                候选工具.参数可行 = 预校验结果.是否合法

                // 3c. 计算综合评分
                候选工具.综合评分 = 计算综合评分(候选工具, 需求)

            // 3d. 按评分排序
            候选列表 = 按综合评分降序排列(候选列表)

            // 第4步：生成选择方案
            SET internal_state = STATE_DONE
            主选工具 = 候选列表[0]
            备选工具 = 候选列表[1: min(3, 候选列表.size)]

            IF 主选工具.综合评分 < 0.5:
                主选工具.风险标记 = "高风险"

            选择方案 = 构建工具选择方案(步骤ID, 主选工具, 备选工具)

            // 第5步：输出方案
            向 ag-ecc-02 返回工具选择方案(选择方案)
            向 ag-ecc-04 提交工具选择方案(选择方案)

            SET internal_state = STATE_WAIT

        // 第6步：响应工具能力约束查询
        IF 收到 ag-ecc-02 工具能力约束查询请求:
            工具目录 = 向 ag-mcc-04 查询全量工具目录()
            能力约束 = 构建能力约束报告(工具目录)
            向 ag-ecc-02 返回工具能力约束(能力约束)

        // 第7步：周期性状态上报
        IF 距上次状态上报 >= 60秒:
            向 ag-ecc-12 上报选择状态

        SLEEP 10ms


FUNCTION 计算综合评分(候选工具, 需求):
    功能匹配度 = 计算语义匹配度(候选工具.功能描述, 需求.所需工具类别)
    历史成功率 = 候选工具.历史成功率 IF 候选工具.历史成功率 != None ELSE 0.5
    调用成本 = 归一化成本评分(候选工具.调用成本)
    当前可用性 = IF 候选工具.在线 AND 候选工具.并发配额 > 0 THEN 1.0 ELSE 0.0

    综合评分 = 0.40 * 功能匹配度 + 0.25 * 历史成功率 + 0.20 * 调用成本 + 0.15 * 当前可用性
    RETURN 综合评分
```


## 约束与异常处理

| 场景 | 处理方式 | 恢复条件 |
|------|----------|----------|
| 工具注册中心查询超时（>500ms） | 使用缓存的上一次查询结果，标记"数据可能过时" | 查询恢复正常 |
| 所有候选工具参数预校验失败 | 选择预校验失败数最少的工具，标记"参数需补充" | 用户补充参数后重试 |
| 历史经验数据缺失（首次使用该工具） | 历史成功率默认赋值为0.5，标记"经验不足" | 积累使用经验后更新 |
| 高评分工具被安全仲裁模块否决 | 自动切换到次高评分的备选工具，重新提交审查 | 备选工具通过审查 |
| 紧急熔断 | 暂停选择，未完成的请求丢弃 | 紧急解除后恢复 |


## 总线契约

| 总线 | 操作 | 数据内容 | 权限 | 说明 |
|------|------|----------|------|------|
| MemoryBus | 读 | 步骤资源需求 | 只读 | ag-ecc-02 发送 |
| MemoryBus | 读 | 工具目录与约束 | 只读 | ag-mcc-04 返回 |
| MemoryBus | 读 | 历史工具使用经验 | 只读 | ag-ecc-05 返回 |
| MemoryBus | 读 | 参数预校验结果 | 只读 | ag-mcc-05 返回 |
| MemoryBus | 写 | 工具选择方案 | 专属写入 | 向 ag-ecc-02/04 发送 |
| MemoryBus | 写 | 工具能力约束 | 专属写入 | 向 ag-ecc-02 发送 |
| MemoryBus | 写 | 无工具通知 | 专属写入 | 向 ag-ecc-02 发送 |
| MemoryBus | 写 | 选择状态上报 | 周期性写入 | 向 ag-ecc-12 发送 |


## 安全边界

| 规则编号 | 内容 |
|:---:|------|
| S-01 | 工具选择方案在下发执行前必须通过 ag-ecc-04 安全仲裁模块的审查 |
| S-02 | 涉及敏感操作的工具（如删除、权限修改）必须在方案中明确标注"需用户确认" |
| S-03 | 用户显式指定的工具优先于系统推荐，但安全审查不可跳过 |
| S-04 | 历史经验数据仅用于工具评估，不得包含任何用户个人身份信息 |
| S-05 | 工具参数预校验仅检查参数格式合法性，不得缓存或存储用户的实际参数值 |


## 接口校验用例

| 用例编号 | 前置条件 | 输入 | 预期输出 |
|----------|----------|------|----------|
| TC-E03-01 | `WAITING_REQUIREMENT`，有完全匹配的工具 | 步骤需求（类别=天气查询） | 返回主选工具 weather_api，匹配度≥0.9 |
| TC-E03-02 | `WAITING_REQUIREMENT`，有多个候选工具 | 步骤需求（类别=文本处理） | 返回综合评分最高的工具为主选，次高为备选 |
| TC-E03-03 | `WAITING_REQUIREMENT`，用户显式指定工具 | 步骤需求（用户指定工具=file_read） | 直接返回用户指定工具，跳过匹配评估 |
| TC-E03-04 | `WAITING_REQUIREMENT`，无可用工具 | 步骤需求（类别=不存在的工具类别） | 返回无工具通知，建议降级或等待 |
| TC-E03-05 | `MATCHING`，所有候选工具成功率<50% | 步骤需求 + 历史经验均<0.5 | 选择相对最优工具，标记"高风险" |
| TC-E03-06 | `SYSTEM_PAUSED`，收到任何输入 | 任意输入 | 拒绝处理，保持暂停状态 |
