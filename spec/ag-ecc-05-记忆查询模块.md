# ag-ecc-05-记忆查询模块 接口规格

---

## 基本信息

| 项 | 内容 |
|----|------|
| 模块编号 | ag-ecc-05 |
| 模块名称 | 记忆查询模块 |
| 所属分区 | 一、认知大脑核心模块 |
| 核心职责 | 作为 ECC 认知大脑与 MLNF-Mem 记忆中枢之间的统一查询网关，负责将认知模块（意图解析、任务规划、工具选择等）的记忆检索需求转化为标准化的查询请求，通过 MemoryBus 发送至 ag-mem-01（总控漏斗F₀）进行双漏斗检索。将记忆中枢返回的原始经验数据转化为认知模块可直接消费的结构化摘要。同时管理查询缓存、结果合并与去重，优化高频查询的响应速度。不参与记忆的写入或管理，仅执行记忆的检索与结果适配 |
| 依赖模块 | ag-mem-01（总控漏斗F₀，通过 MemoryBus 转发查询至双漏斗记忆系统） |
| 被依赖模块 | ag-ecc-01（意图解析模块，查询用户历史偏好）、ag-ecc-02（任务规划模块，查询历史任务模板）、ag-ecc-03（工具选择模块，查询工具使用经验）、ag-ecc-08（元认知模块，查询失败经验）、ag-ecc-10（社会心智模块，查询交互历史） |


## 内部状态定义

| 状态 | 标识 | 含义 | 触发条件 |
|------|------|------|----------|
| 等待查询 | `WAITING_QUERY` | 无待处理查询，等待上游模块请求 | 系统初始化完成，查询队列为空 |
| 查询处理中 | `QUERYING` | 已向记忆中枢发送查询，等待结果返回 | 收到上游模块的记忆查询请求 |
| 结果整理中 | `FORMATTING` | 记忆中枢已返回原始数据，正在整理为结构化摘要 | 收到 ag-mem-01 的查询回执 |
| 查询超时 | `QUERY_TIMEOUT` | 记忆中枢响应超时，尝试返回缓存或降级结果 | 查询等待时间超过阈值 |
| 暂停服务 | `SYSTEM_PAUSED` | 系统紧急熔断 | 收到紧急熔断指令 |

## 查询缓存策略

| 缓存类型 | 缓存时长 | 最大条目数 | 淘汰策略 | 说明 |
|----------|:---:|:---:|------|------|
| 用户偏好摘要 | 300秒（5分钟） | 50条 | LRU | 同一会话内偏好变化缓慢 |
| 历史任务模板 | 600秒（10分钟） | 100条 | LRU | 任务模板相对稳定 |
| 工具使用经验 | 120秒（2分钟） | 200条 | LRU | 工具经验更新较快 |
| 失败经验 | 不缓存 | — | — | 失败经验需实时反映最新状态 |
| 交互历史 | 60秒（1分钟） | 30条 | LRU | 交互历史更新频繁 |


## 核心处理逻辑

```
FUNCTION memory_query_main_loop():
    STATE_WAIT = WAITING_QUERY
    STATE_QUERY = QUERYING
    STATE_FORMAT = FORMATTING
    STATE_TIMEOUT = QUERY_TIMEOUT
    STATE_PAUSED = SYSTEM_PAUSED

    SET internal_state = STATE_WAIT
    初始化查询缓存 = {用户偏好: {}, 任务模板: {}, 工具经验: {}, 交互历史: {}}
    初始化活跃查询表 = {}  // 查询ID → {请求, 开始时间, 来源模块}

    WHILE 系统运行中:
        // 第1步：紧急熔断
        IF 收到紧急熔断指令:
            SET internal_state = STATE_PAUSED
            CONTINUE
        ELSE IF 收到恢复指令 AND internal_state == STATE_PAUSED:
            SET internal_state = STATE_WAIT

        // 第2步：接收记忆中枢查询回执
        IF 收到 ag-mem-01 查询回执:
            查询ID = 回执.查询ID
            IF 查询ID IN 活跃查询表:
                请求信息 = 活跃查询表[查询ID]
                SET internal_state = STATE_FORMAT
                整理结果 = 根据请求类型整理回执(请求信息.请求, 回执)
                向 请求信息.来源模块 发送整理结果
                从活跃查询表中移除查询ID
                SET internal_state = STATE_WAIT
                CONTINUE

        // 第3步：检查查询超时
        FOR EACH 查询ID, 请求信息 IN 活跃查询表:
            IF NOW() - 请求信息.开始时间 > 超时阈值:
                SET internal_state = STATE_TIMEOUT
                尝试返回缓存结果(请求信息)
                向 请求信息.来源模块 发送查询超时通知
                从活跃查询表中移除查询ID
        IF internal_state == STATE_TIMEOUT:
            SET internal_state = STATE_WAIT

        // 第4步：接收上游模块查询请求
        IF 收到查询请求:
            // 4a. 检查缓存
            缓存键 = 生成缓存键(请求)
            IF 请求类型 != "失败经验" AND 缓存命中:
                直接返回缓存结果
                CONTINUE

            // 4b. 构建标准化查询并发送至记忆中枢
            SET internal_state = STATE_QUERY
            查询ID = 生成查询ID()
            活跃查询表[查询ID] = {请求: 请求, 开始时间: NOW(), 来源模块: 请求.来源模块}

            标准化查询 = 转化为标准化查询(请求)
            向 ag-mem-01 发送查询请求(查询ID, 标准化查询)

        // 第5步：周期性状态上报
        IF 距上次状态上报 >= 60秒:
            向 ag-ecc-12 上报查询状态

        SLEEP 10ms


FUNCTION 根据请求类型整理回执(请求, 回执):
    CASE 请求.请求类型 OF:
        "用户偏好":
            RETURN 构建用户偏好摘要(回执.匹配经验列表)
        "历史任务模板":
            RETURN 构建历史任务模板(回执.匹配经验列表)
        "工具经验":
            RETURN 构建工具使用经验摘要(回执.匹配经验列表)
        "失败经验":
            RETURN 构建失败经验列表(回执.匹配经验列表)
        "交互历史":
            RETURN 构建交互历史摘要(回执.匹配经验列表)
```


## 约束与异常处理

| 场景 | 处理方式 | 恢复条件 |
|------|----------|----------|
| 记忆中枢查询超时（>2秒） | 尝试返回缓存结果，若无缓存则返回查询超时通知 | 记忆中枢恢复正常 |
| 记忆中枢返回空结果 | 返回空结果 + 标注"无匹配经验"，不报错 | — |
| 记忆中枢返回部分结果（某漏斗不可用） | 使用可用漏斗的数据返回，标注"结果不完整" | 故障漏斗恢复 |
| 查询请求参数不完整（缺少用户ID或关键词） | 拒绝查询，返回错误提示 | 上游补充参数后重试 |
| 缓存满（达上限） | 按 LRU 淘汰最久未访问的缓存条目 | — |
| 紧急熔断 | 清空活跃查询表，暂停所有查询 | 紧急解除后恢复 |

## 安全边界

| 规则编号 | 内容 |
|:---:|------|
| Q-01 | 本模块仅读取记忆数据，不参与任何记忆写入、修改或删除操作 |
| Q-02 | 查询请求不得包含用户的原始输入内容，仅发送结构化的查询条件（关键词、任务类型、时间窗口） |
| Q-03 | 返回的记忆数据在传递至上游模块前必须经过脱敏处理：用户ID替换为匿名标识，原始输入文本仅返回特征摘要 |
| Q-04 | 查询缓存仅存储脱敏后的摘要数据，不得缓存完整的经验原始数据 |
| Q-05 | 不同用户之间的查询缓存必须隔离，禁止跨用户缓存命中 |


## 接口校验用例

| TC-E05-01 | `WAITING_QUERY`，正常用户偏好查询 | 查询请求（类型=用户偏好，用户ID=U001） | 向 ag-mem-01 发送查询，等待回执，返回偏好摘要 |
