# ag-ecc-09-内生动机模块 接口规格

---

## 基本信息

| 项 | 内容 |
|----|------|
| 模块编号 | ag-ecc-09 |
| 模块名称 | 内生动机模块 |
| 所属分区 | 一、认知大脑核心模块 |
| 核心职责 | 基于系统对自身能力短板、环境变化、用户长期偏好以及未完成任务的持续感知，自主生成探索、学习与任务执行的内在驱动力。将认知缺口、未知领域或优化机会转化为具体的“内生目标”，提交至 ag-ecc-02（任务规划模块）进行编排执行。驱动系统从被动响应工具向主动成长型智能体演化。不参与目标的具体执行，仅负责目标的自主发现、优先级评估与生成 |
| 依赖模块 | ag-ecc-02（任务规划模块，提交自主生成的目标）、ag-ecc-05（记忆查询模块，获取用户长期偏好与未完成任务）、ag-ecc-08（元认知模块，接收认知偏差告警与自主学习需求）、ag-ecc-12（资源调度模块，申请探索行为所需资源） |
| 被依赖模块 | ag-ecc-02（接收自主目标）、ag-ecc-08（接收自主学习需求反馈） |


## 内部状态定义

| 状态 | 标识 | 含义 | 触发条件 |
|------|------|------|----------|
| 休眠 | `DORMANT` | 无明确的自主目标生成需求，系统处于被动响应模式 | 系统初始化完成，资源充足且用户任务繁忙 |
| 环境感知 | `ENVIRONMENT_SCAN` | 正在主动扫描环境变化、能力缺口与优化机会 | 系统空闲资源充足，或收到元认知的学习需求 |
| 目标生成 | `GOAL_GENERATION` | 正在基于感知结果生成具体的自主探索或学习目标 | 环境扫描发现机会点或能力缺口 |
| 目标排队 | `GOAL_QUEUING` | 已生成多个候选目标，正在按优先级排序与冲突消解 | 目标生成完成，存在多个候选目标 |
| 暂停服务 | `SYSTEM_PAUSED` | 系统紧急熔断 | 收到紧急熔断指令 |


## 输入数据

| 输入项 | 数据类型 | 来源模块 | 触发条件 | 优先级 |
|--------|----------|----------|----------|:---:|
| 认知偏差告警 | Struct（偏差类型 + 涉及能力维度 + 严重度 + 建议纠正措施） | ag-ecc-08 元认知模块 | 检测到显著认知偏差时 | **高** |
| 自主学习需求 | Struct（能力缺口描述 + 目标能力维度 + 建议学习方式 + 优先级） | ag-ecc-08 元认知模块 | 能力评估发现短板时 | **高** |
| 用户长期偏好摘要 | Struct（偏好关键词 + 高频任务类型 + 兴趣趋势 + 未完成目标列表） | ag-ecc-05 记忆查询模块 | 环境扫描时查询 | 普通 |
| 系统资源状态 | Struct（CPU使用率 + 内存使用率 + 存储空间 + 大模型调用配额剩余） | ag-ecc-12 资源调度模块 | 生成目标前查询 | **高** |
| 用户当前活动状态 | Struct（是否在线 + 当前活跃会话数 + 是否有未读消息） | 系统会话管理器 | 环境扫描时查询 | 普通 |
| 全局调度指令 | Enum（暂停/恢复/熔断） | ag-ecc-12 资源调度模块 | 模式切换或紧急事件时 | **紧急** |


## 输出数据

| 输出项 | 数据类型 | 目标模块 | 输出条件 | 优先级 |
|--------|----------|----------|----------|:---:|
| 自主任务目标 | Struct（目标ID + 目标类型：学习/探索/维护/建议 + 目标描述 + 预期收益 + 预估资源消耗 + 优先级 + 生成原因） | ag-ecc-02 任务规划模块 | 目标生成完成并通过优先级筛选时 | 普通（低于用户任务） |
| 主动建议（推送给用户） | Struct（建议ID + 建议内容 + 触发原因 + 用户可选操作） | 用户交互层 | 生成高价值的用户侧建议时 | 普通 |
| 学习任务完成反馈 | Struct（学习目标ID + 完成状态 + 能力提升评估） | ag-ecc-08 元认知模块 | 自主发起的任务完成时 | 普通 |
| 动机状态上报 | Struct（当前状态 + 已生成目标数 + 排队中目标数 + 累计完成数 + 最近活跃时间） | ag-ecc-12 资源调度模块 | 周期性（每120秒）或状态变更时 | 普通 |


## 内生目标类型

| 目标类型 | 说明 | 典型触发条件 | 优先级范围 |
|----------|------|------|:---:|
| 能力补全 | 针对元认知识别的能力短板，主动发起学习 | 认知偏差告警或自主学习需求 | 中（0.5-0.7） |
| 环境探索 | 发现新工具、新数据源或未知领域，尝试探索 | 工具注册中心新增工具、环境变化 | 低（0.3-0.5） |
| 记忆维护 | 清理冗余记忆、整理知识图谱、优化检索效率 | 记忆中枢容量预警、碎片化指数高 | 低（0.2-0.4） |
| 用户服务优化 | 基于用户长期偏好，主动生成建议或预准备 | 用户偏好变化趋势、重复性任务模式 | 中（0.5-0.6） |
| 自我反思 | 复盘失败任务，总结教训，优化内部规则 | 失败率超过阈值 | 高（0.7-0.9） |


## 核心处理逻辑

```
FUNCTION intrinsic_motivation_main_loop():
    STATE_DORMANT = DORMANT
    STATE_SCAN = ENVIRONMENT_SCAN
    STATE_GENERATE = GOAL_GENERATION
    STATE_QUEUE = GOAL_QUEUING
    STATE_PAUSED = SYSTEM_PAUSED

    SET internal_state = STATE_DORMANT
    初始化候选目标队列 = []
    初始化目标冷却期表 = {}  // 目标类型 → 上次生成时间

    WHILE 系统运行中:
        // 第1步：紧急熔断
        IF 收到紧急熔断指令:
            SET internal_state = STATE_PAUSED
            CONTINUE
        ELSE IF 收到恢复指令 AND internal_state == STATE_PAUSED:
            SET internal_state = STATE_DORMANT

        // 第2步：接收元认知的驱动信号
        IF 收到认知偏差告警 OR 自主学习需求:
            SET internal_state = STATE_GENERATE

            // 立即生成对应的能力补全目标
            IF 收到认知偏差告警:
                目标 = 生成能力补全目标(偏差告警)
            ELSE:
                目标 = 生成能力补全目标(学习需求)

            // 检查冷却期
            IF 目标.类型 NOT IN 目标冷却期表 OR NOW() - 目标冷却期表[目标.类型] > 冷却间隔:
                候选目标队列.按优先级插入(目标)
                目标冷却期表[目标.类型] = NOW()

            SET internal_state = STATE_DORMANT
            CONTINUE

        // 第3步：环境感知扫描
        IF internal_state == STATE_DORMANT:
            系统资源 = 向 ag-ecc-12 查询系统资源状态()

            // 仅当系统资源充足且用户不繁忙时触发主动扫描
            IF 系统资源.CPU使用率 < 50% AND 系统资源.内存使用率 < 70%:
                用户状态 = 查询用户当前活动状态()
                IF 用户状态.当前活跃会话数 < 3:
                    SET internal_state = STATE_SCAN

        // 第4步：目标生成
        IF internal_state == STATE_SCAN:
            SET internal_state = STATE_GENERATE

            // 4a. 扫描用户长期偏好
            用户偏好 = 向 ag-ecc-05 查询用户长期偏好摘要()
            IF 用户偏好.未完成目标列表 非空:
                FOR EACH 未完成目标 IN 用户偏好.未完成目标列表:
                    提醒目标 = 生成用户提醒目标(未完成目标)
                    候选目标队列.按优先级插入(提醒目标)

            // 4b. 扫描环境变化与探索机会
            IF 检测到新工具可用():
                探索目标 = 生成探索目标(新工具信息)
                候选目标队列.按优先级插入(探索目标)

            // 4c. 扫描记忆健康状态
            IF 记忆中枢碎片化指数 > 阈值:
                维护目标 = 生成记忆维护目标()
                候选目标队列.按优先级插入(维护目标)

            SET internal_state = STATE_QUEUE

        // 第5步：目标排队与下发
        IF internal_state == STATE_QUEUE:
            // 过滤低收益目标
            候选目标队列 = 过滤低收益目标(候选目标队列)

            // 限制同时活跃的自主目标数量（最多3个）
            WHILE 候选目标队列 非空 AND 活跃自主目标数 < 3:
                最优目标 = 候选目标队列.pop(0)
                向 ag-ecc-02 发送自主任务目标(最优目标)
                记录动机状态

            SET internal_state = STATE_DORMANT

        // 第6步：处理学习任务完成反馈
        IF 收到学习任务完成反馈:
            向 ag-ecc-08 转发学习任务完成反馈(反馈)

        // 第7步：周期性状态上报
        IF 距上次状态上报 >= 120秒:
            向 ag-ecc-12 上报动机状态

        SLEEP 200ms
```


## 约束与异常处理

| 场景 | 处理方式 | 恢复条件 |
|------|----------|----------|
| 系统资源不足（CPU/内存占用高） | 暂停扫描与目标生成，进入 DORMANT 状态 | 资源占用降至阈值以下 |
| 候选目标队列积压超过 10 个 | 按优先级强制排序，丢弃最低优先级的 3 个目标 | 队列消化至正常水平 |
| 同一类型目标在冷却期内重复触发 | 忽略，等待冷却期结束 | 冷却期结束 |
| 用户在线且当前有紧急任务 | 暂停所有自主目标生成，优先保障用户任务资源 | 用户空闲或任务完成 |
| 紧急熔断 | 清空候选目标队列，暂停所有目标生成活动 | 紧急解除后恢复 |

## 安全边界

| 规则编号 | 内容 |
|:---:|------|
| I-01 | 自主生成的目标不得涉及安全敏感操作（系统配置修改、权限变更、数据删除等） |
| I-02 | 用户在线且有活跃任务时，自主目标的优先级永远低于用户显式任务 |
| I-03 | 同一类型的自主目标在冷却期内不得重复生成，防止系统陷入无意义的循环 |
| I-04 | 自主目标在下发执行前必须通过 ag-ecc-04 安全仲裁模块的审查 |
| I-05 | 系统资源不足时，禁止生成任何需要调用大模型或占用大量计算资源的探索目标 |


## 接口校验用例

| 用例编号 | 前置条件 | 输入 | 预期输出 |
|----------|----------|------|----------|
| TC-E09-01 | `DORMANT`，收到认知偏差告警 | 认知偏差告警（类型=过度自信，维度=意图理解） | 进入GOAL_GENERATION→生成能力补全目标→插入候选队列 |
| TC-E09-02 | `DORMANT`，系统空闲触发环境扫描 | 资源充足，用户不繁忙 | 进入ENVIRONMENT_SCAN→扫描偏好/工具/记忆→生成探索目标 |
| TC-E09-03 | `GOAL_GENERATION`，检测到新工具 | 工具注册中心新增工具通知 | 生成探索目标（优先级0.3-0.5），插入候选队列 |
| TC-E09-04 | `GOAL_QUEUING`，候选队列超过10个 | 候选队列积压 | 强制排序，丢弃最低优先级3个目标 |
| TC-E09-05 | `DORMANT`，同一类型目标在冷却期内 | 重复触发能力补全目标（距上次生成不足冷却间隔） | 忽略，不生成重复目标 |
| TC-E09-06 | `SYSTEM_PAUSED`，收到任何驱动信号 | 认知偏差告警 | 拒绝处理，保持暂停状态 |
