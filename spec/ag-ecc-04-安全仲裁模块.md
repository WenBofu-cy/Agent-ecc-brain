# ag-ecc-04-安全仲裁模块 接口规格

---

## 基本信息

| 项 | 内容 |
|----|------|
| 模块编号 | ag-ecc-04 |
| 模块名称 | 安全仲裁模块 |
| 所属分区 | 一、认知大脑核心模块 |
| 核心职责 | 作为 ECC 认知大脑的全局安全审查与仲裁中枢，对 ag-ecc-02（任务规划模块）输出的任务执行计划和 ag-ecc-03（工具选择模块）输出的工具选择方案进行安全合规性终审。基于安全规则库（ag-mem-45）、操作风险等级与系统当前安全态势，判定每项操作是否允许执行、需要用户确认或直接拒绝。拥有全局最高否决权——任何模块的决策均可被本模块拦截。在检测到高风险操作或安全态势恶化时，可触发系统降级、熔断或紧急闭锁。同时负责安全事件审计日志的记录与维护 |
| 依赖模块 | ag-ecc-02（任务规划模块，接收任务执行计划供审查）、ag-ecc-03（工具选择模块，接收工具选择方案供审查）、ag-mem-45（安全规则库，查询安全规则与合规基准）、ag-ecc-05（记忆查询模块，查询历史安全事件）、ag-ecc-12（资源调度模块，下发熔断/降级指令） |
| 被依赖模块 | ag-ecc-02（返回审查结果）、ag-ecc-03（返回审查结果）、ag-ecc-12（接收熔断/降级指令）、ag-mem-51（记忆变更日志追溯单元，记录安全事件） |


## 内部状态定义

| 状态 | 标识 | 含义 | 触发条件 |
|------|------|------|----------|
| 正常监控 | `NORMAL_MONITOR` | 安全态势正常，执行常规审查 | 系统初始化完成，安全态势评分 ≥ 0.7 |
| 加强审查 | `ENHANCED_REVIEW` | 安全态势偏低，所有操作需额外确认 | 安全态势评分在 0.4 – 0.7 之间，或近期有安全事件 |
| 高风险警戒 | `HIGH_ALERT` | 安全态势严峻，仅允许最低风险操作 | 安全态势评分 < 0.4，或检测到持续攻击 |
| 紧急闭锁 | `EMERGENCY_LOCKDOWN` | 系统全局闭锁，拒绝一切非安全操作 | 检测到严重安全威胁或收到人工闭锁指令 |
| 暂停服务 | `SYSTEM_PAUSED` | 系统紧急熔断 | 收到紧急熔断指令 |

## 安全审查规则

### 一、操作风险分级

| 风险等级 | 标识 | 判定条件 | 审查策略 |
|:---:|:---:|------|------|
| 无风险 | `SAFE` | 工具在白名单中，操作类型为只读，无敏感参数 | 自动放行 |
| 低风险 | `LOW` | 工具在白名单中，操作类型为写入但目标非关键系统 | 自动放行，记录日志 |
| 中风险 | `MEDIUM` | 工具不在白名单但也不在黑名单，或涉及用户数据读取 | 需用户确认 |
| 高风险 | `HIGH` | 工具在敏感操作清单中，或涉及隐私数据、系统配置修改 | 需用户二次确认 + 附加安全约束 |
| 严重风险 | `CRITICAL` | 工具在黑名单中，或操作类型为删除、权限变更、系统级命令 | 直接拒绝 |
| 极端风险 | `EXTREME` | 检测到攻击模式（合规模式匹配命中），或工具调用序列触发告警规则 | 触发紧急闭锁 |

### 二、审查结论

| 结论 | 标识 | 含义 | 后续动作 |
|:---:|:---:|------|------|
| 放行 | `APPROVED` | 审查通过，可以执行 | 下发至执行层 |
| 需确认 | `CONFIRM_REQUIRED` | 需要用户明确确认后方可执行 | 向用户发送确认请求，等待回复 |
| 拒绝 | `REJECTED` | 审查不通过，禁止执行 | 向请求来源返回拒绝原因，记录安全事件 |
| 降级执行 | `DEGRADED_APPROVED` | 审查通过，但需在安全约束下执行（如限速、沙箱隔离） | 下发时附带安全约束参数 |


## 核心处理逻辑

```
FUNCTION safety_arbiter_main_loop():
    STATE_NORMAL = NORMAL_MONITOR
    STATE_ENHANCED = ENHANCED_REVIEW
    STATE_HIGH = HIGH_ALERT
    STATE_LOCKDOWN = EMERGENCY_LOCKDOWN
    STATE_PAUSED = SYSTEM_PAUSED

    SET internal_state = STATE_NORMAL
    初始化安全态势评分 = 0.85
    初始化近期安全事件列表 = []

    WHILE 系统运行中:
        // 第1步：紧急熔断
        IF 收到紧急熔断指令:
            SET internal_state = STATE_PAUSED
            CONTINUE
        ELSE IF 收到恢复指令 AND internal_state == STATE_PAUSED:
            SET internal_state = STATE_NORMAL

        // 第2步：处理人工安全指令（最高优先级）
        IF 收到人工安全指令:
            校验授权令牌(指令.授权令牌)
            IF 令牌无效:
                记录安全事件("未授权的人工干预尝试")
                CONTINUE

            CASE 指令.指令类型 OF:
                "强制闭锁":
                    SET internal_state = STATE_LOCKDOWN
                    向 ag-ecc-12 发送安全降级/熔断指令("闭锁", 指令.原因)
                "解除闭锁":
                    IF internal_state == STATE_LOCKDOWN:
                        SET internal_state = STATE_NORMAL
                        重新评估安全态势()
                "强制放行":
                    向请求来源返回安全审查结果(APPROVED, "人工强制放行")
                "强制拒绝":
                    向请求来源返回安全审查结果(REJECTED, "人工强制拒绝")

        // 第3步：评估安全态势
        安全态势评分 = 计算安全态势(近期安全事件列表)
        IF 安全态势评分 < 0.4 AND internal_state != STATE_LOCKDOWN:
            SET internal_state = STATE_HIGH
        ELSE IF 安全态势评分 < 0.7 AND internal_state == STATE_NORMAL:
            SET internal_state = STATE_ENHANCED
        ELSE IF 安全态势评分 >= 0.7 AND internal_state IN (STATE_ENHANCED, STATE_HIGH):
            SET internal_state = STATE_NORMAL

        // 第4步：接收待审查的计划/方案
        IF 收到任务执行计划 OR 工具选择方案:
            // 紧急闭锁状态下直接拒绝
            IF internal_state == STATE_LOCKDOWN:
                向请求来源返回安全审查结果(REJECTED, "系统处于紧急闭锁状态")
                CONTINUE

            // 加载安全规则
            安全规则 = 向 ag-mem-45 查询安全规则与合规基准()

            // 判定风险等级
            IF 输入类型 == "任务执行计划":
                风险等级 = 评估任务计划风险(任务执行计划, 安全规则)
            ELSE:
                风险等级 = 评估工具方案风险(工具选择方案, 安全规则)

            // 根据风险等级和安全态势生成审查结论
            审查结论 = 生成审查结论(风险等级, internal_state)

            // 输出结果
            CASE 审查结论 OF:
                APPROVED:
                    向请求来源返回安全审查结果(APPROVED)
                CONFIRM_REQUIRED:
                    向用户发送确认请求(操作描述, 风险说明)
                    等待用户确认超时=60秒
                    IF 用户确认:
                        向请求来源返回安全审查结果(APPROVED, "用户已确认")
                    ELSE:
                        向请求来源返回安全审查结果(REJECTED, "用户未确认或超时")
                REJECTED:
                    向请求来源返回安全审查结果(REJECTED, 拒绝原因)
                DEGRADED_APPROVED:
                    附加约束 = 生成安全约束(风险等级)
                    向请求来源返回安全审查结果(DEGRADED_APPROVED, 附加约束)

            // 记录安全事件日志
            向 ag-mem-51 发送安全事件日志(审查ID, 审查结论, 风险等级)

        // 第5步：检测到极端风险时触发紧急闭锁
        IF 风险等级 == EXTREME AND internal_state != STATE_LOCKDOWN:
            SET internal_state = STATE_LOCKDOWN
            向 ag-ecc-12 发送安全降级/熔断指令("紧急闭锁", "检测到攻击模式")

        // 第6步：周期性安全态势报告
        IF 距上次态势报告 >= 60秒:
            向 ag-ecc-12 发送安全态势报告

        SLEEP 10ms


FUNCTION 评估工具方案风险(方案, 安全规则):
    // 黑名单检查
    IF 方案.主选工具.名称 IN 安全规则.黑名单:
        RETURN CRITICAL

    // 白名单检查
    IF 方案.主选工具.名称 IN 安全规则.白名单:
        IF 方案.操作类型 == "只读":
            RETURN SAFE
        RETURN LOW

    // 敏感操作检查
    IF 方案.主选工具.名称 IN 安全规则.敏感操作清单:
        RETURN HIGH

    // 合规模式匹配
    IF 匹配合规告警模式(方案.工具调用序列, 安全规则.合规模式):
        RETURN EXTREME

    RETURN MEDIUM


FUNCTION 生成审查结论(风险等级, 系统状态):
    IF 系统状态 == STATE_HIGH AND 风险等级 >= MEDIUM:
        RETURN REJECTED

    IF 系统状态 == STATE_ENHANCED AND 风险等级 >= HIGH:
        RETURN CONFIRM_REQUIRED

    SWITCH 风险等级:
        CASE SAFE: RETURN APPROVED
        CASE LOW: RETURN APPROVED
        CASE MEDIUM: RETURN CONFIRM_REQUIRED
        CASE HIGH: RETURN CONFIRM_REQUIRED
        CASE CRITICAL: RETURN REJECTED
        CASE EXTREME: RETURN REJECTED
```


## 约束与异常处理

| 场景 | 处理方式 | 恢复条件 |
|------|----------|----------|
| 安全规则库不可用（查询超时或返回空） | 使用编译期内置的最小安全规则集，所有操作自动升级一个风险等级 | 安全规则库恢复正常 |
| 用户确认超时（>60秒） | 自动拒绝，返回"用户未确认" | 用户重新发起操作 |
| 人工安全指令授权令牌无效 | 拒绝指令，记录"未授权的人工干预尝试" | 使用有效令牌重试 |
| 安全态势评分连续下降 | 自动提升审查等级，触发安全态势报告 | 安全事件减少，评分回升 |
| 紧急闭锁状态收到人工解锁指令 | 校验授权令牌后恢复至 NORMAL_MONITOR，重新评估态势 | 令牌有效且系统安全 |
| 紧急熔断 | 暂停所有审查活动，保持当前安全态势状态 | 紧急解除后恢复 |

## 安全边界

| 规则编号 | 内容 |
|:---:|------|
| A-01 | 本模块为 ECC 认知大脑的最高安全决策单元，拥有全局最高否决权，任何模块不得绕过本模块执行操作 |
| A-02 | 安全规则库不可用时，使用编译期内置最小安全规则集，采用保守策略（所有操作风险+1级） |
| A-03 | 人工安全指令必须经过授权令牌验证与双重确认，每次干预必须完整记录不可变日志 |
| A-04 | 紧急闭锁状态下，仅接收人工解锁指令，一切自动化操作请求被直接拒绝 |
| A-05 | 用户确认请求的超时时间为 60 秒，超时自动拒绝，防止无限等待 |
| A-06 | 安全事件日志必须完整记录每次审查的输入、规则匹配、结论与时间戳，不可篡改 |
