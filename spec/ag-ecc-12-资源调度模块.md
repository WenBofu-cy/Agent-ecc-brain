# ag-ecc-12-资源调度模块 接口规格

---

## 基本信息

| 项 | 内容 |
|----|------|
| 模块编号 | ag-ecc-12 |
| 模块名称 | 资源调度模块 |
| 所属分区 | 一、认知大脑核心模块 |
| 核心职责 | 作为 ECC 认知大脑的全局中枢与唯一对外网关，统筹管理系统的算力、内存、存储、网络带宽及大模型 API 调用配额等关键资源。负责跨系统通信的统一代理——所有 ECC 模块与记忆中枢（MLNF-Mem）的 MemoryBus 通信、与运动小脑（MCC）的 CerebellumBus 通信、与外部大模型服务的 API 调用，均需通过本模块中转与安全校验。同时承担系统熔断保护、人机主权闭锁、权限管控、全链路审计与资源调度日志的归档。不参与任何认知决策，仅执行资源分配、通信代理与安全策略的执行 |
| 依赖模块 | 全部 ag-ecc-01 至 ag-ecc-11 模块（资源申请、异常告警、状态上报均汇总至本模块）、ag-mem-01（总控漏斗F₀，通过 MemoryBus 转发记忆查询/写入请求）、ag-mcc-01（执行调度核心，通过 CerebellumBus 下发工具调用指令）、外部大模型 API 服务 |
| 被依赖模块 | 全部 ag-ecc-01 至 ag-ecc-11 模块（接收资源配额、权限管控、对外通信代理、熔断信号） |


## 内部状态定义

| 状态 | 标识 | 含义 | 触发条件 |
|------|------|------|----------|
| 正常调度 | `NORMAL_SCHEDULING` | 系统资源充足，正常进行资源分配与通信代理 | 系统初始化完成，资源使用率在安全范围内 |
| 资源预警 | `RESOURCE_WARNING` | 某项关键资源使用率超过预警阈值 | CPU/内存/存储使用率 ≥ 80% |
| 资源紧张 | `RESOURCE_CRITICAL` | 系统资源极度紧张，启动限流与降级 | CPU/内存/存储使用率 ≥ 95% |
| 熔断保护 | `CIRCUIT_BREAKER` | 系统触发熔断，拒绝所有非安全操作 | 收到安全仲裁模块的紧急熔断指令，或系统连续异常 |
| 主权闭锁 | `HUMAN_LOCKDOWN` | 人机主权闭锁，所有自动化决策暂停，全权移交人类 | 检测到高危操作、人工触发或安全态势极度恶化 |
| 暂停服务 | `SYSTEM_PAUSED` | 系统紧急熔断 | 收到紧急熔断指令 |

## 资源配额与限流规则

### 一、资源监控阈值

| 资源类型 | 正常范围 | 预警阈值 | 紧急阈值 | 监控频率 |
|----------|:---:|:---:|:---:|:---:|
| CPU 使用率 | < 70% | ≥ 80% | ≥ 95% | 每秒 |
| 内存使用率 | < 70% | ≥ 80% | ≥ 95% | 每秒 |
| 存储使用率 | < 70% | ≥ 80% | ≥ 95% | 每30秒 |
| 大模型 Token 配额 | < 日限额×70% | ≥ 日限额×80% | ≥ 日限额×95% | 每次调用后 |
| 并发连接数 | < 最大×70% | ≥ 最大×80% | ≥ 最大×95% | 每秒 |

### 二、降级措施

| 降级等级 | 触发条件 | 降级措施 |
|:---:|------|------|
| 一级降级 | 单项资源预警 | 暂停非关键后台任务，延长轮询间隔至500ms |
| 二级降级 | 单项资源紧急或多资源预警 | 暂停大模型调用，限制记忆中枢写入频率 |
| 三级降级 | 多资源紧急或收到熔断指令 | 仅保留安全模块与基本通信，暂停一切非必要服务 |


## 核心处理逻辑

```
FUNCTION resource_scheduler_main_loop():
    STATE_NORMAL = NORMAL_SCHEDULING
    STATE_WARN = RESOURCE_WARNING
    STATE_CRITICAL = RESOURCE_CRITICAL
    STATE_BREAKER = CIRCUIT_BREAKER
    STATE_LOCKDOWN = HUMAN_LOCKDOWN
    STATE_PAUSED = SYSTEM_PAUSED

    SET internal_state = STATE_NORMAL
    初始化资源使用率缓存 = {}
    初始化通信队列 = []
    初始化大模型配额计数器 = 0

    WHILE 系统运行中:
        // 第1步：紧急熔断
        IF 收到紧急熔断指令:
            SET internal_state = STATE_PAUSED
            CONTINUE
        ELSE IF 收到恢复指令 AND internal_state == STATE_PAUSED:
            SET internal_state = STATE_NORMAL

        // 第2步：处理最高优先级的安全与主权指令
        IF 收到人机主权闭锁信号:
            SET internal_state = STATE_LOCKDOWN
            清空通信队列
            向全部 ECC 模块广播主权闭锁确认
            CONTINUE

        IF 收到安全降级/熔断指令:
            SET internal_state = STATE_BREAKER
            向全部 ECC 模块广播熔断通知(指令.目标等级, 指令.触发原因)
            CONTINUE

        // 第3步：资源监控与状态更新
        系统资源 = 采集系统资源监控数据()
        更新资源使用率缓存

        IF 系统资源.任一维度 >= 95%:
            SET internal_state = STATE_CRITICAL
            向全部 ECC 模块广播资源预警通知("紧急", 系统资源)
            触发三级降级()
        ELSE IF 系统资源.任一维度 >= 80% AND internal_state == STATE_NORMAL:
            SET internal_state = STATE_WARN
            向全部 ECC 模块广播资源预警通知("预警", 系统资源)
            触发一级降级()

        // 第4步：处理跨系统通信请求
        IF 收到跨系统通信请求:
            // 熔断或闭锁状态下拒绝所有非安全通信
            IF internal_state IN (STATE_BREAKER, STATE_LOCKDOWN):
                IF 请求.消息类型 NOT IN ("安全指令", "熔断确认"):
                    向请求来源返回拒绝("系统处于熔断/闭锁状态")
                    CONTINUE

            // 根据目标系统路由
            CASE 请求.目标系统 OF:
                "MLNF-Mem":
                    向 ag-mem-01 发送 MemoryBus 转发消息(请求)
                "MCC":
                    向 ag-mcc-01 发送 CerebellumBus 下发指令(请求)
                "大模型":
                    IF 大模型配额计数器 < 日限额:
                        向外部大模型 API 发送请求(请求)
                        大模型配额计数器 += 估算Token消耗(请求)
                    ELSE:
                        向请求来源返回拒绝("大模型日配额已耗尽")

            // 记录审计日志
            向 ag-mem-51 发送全链路审计日志(请求)

        // 第5步：处理资源查询请求
        IF 收到资源查询请求:
            分配结果 = 评估并分配资源(请求)
            向请求来源返回资源分配结果(分配结果)

        // 第6步：周期性状态上报
        IF 距上次状态上报 >= 30秒:
            向系统管理接口上报调度状态

        SLEEP 10ms
```


## 约束与异常处理

| 场景 | 处理方式 | 恢复条件 |
|------|----------|----------|
| 外部系统不可用（MLNF-Mem/MCC/大模型超时） | 向请求来源返回超时错误，记录通信失败日志 | 外部系统恢复，连续3次心跳正常 |
| 大模型日配额耗尽 | 拒绝新的大模型调用请求，返回"配额已耗尽，请明日再试" | 次日零点配额重置 |
| 通信队列积压超过 1000 条 | 优先处理高优先级请求，低优先级请求延迟处理或丢弃 | 队列消化至正常水平 |
| 资源持续紧张且降级无效 | 升级至熔断状态，通知安全仲裁模块 | 资源压力缓解 |
| 主权闭锁期间收到自动化操作请求 | 直接拒绝，返回"系统处于主权闭锁状态" | 人工解除闭锁 |
| 紧急熔断 | 清空通信队列，保留审计日志，暂停所有通信代理 | 紧急解除后恢复 |


## 总线契约

| 总线 | 操作 | 数据内容 | 权限 | 说明 |
|------|------|----------|------|------|
| MemoryBus | 读 | 跨系统通信请求（目标=MLNF-Mem） | 只读 | ag-ecc-01~11 发送 |
| MemoryBus | 写 | MemoryBus 转发消息 | 专属写入 | 向 ag-mem-01 转发 |
| CerebellumBus | 写 | CerebellumBus 下发指令 | 专属写入 | 向 ag-mcc-01 下发 |
| HTTP(S) | 写 | 大模型 API 请求 | 专属写入 | 向外部大模型服务发送 |
| HTTP(S) | 读 | 大模型响应数据 | 只读 | 外部大模型服务返回 |
| MemoryBus | 写 | 资源预警通知 / 限流降级指令 / 熔断广播 / 主权闭锁确认 | 广播 | 向全部 ECC 模块发送 |
| MemoryBus | 写 | 全链路审计日志 | 专属写入 | 向 ag-mem-51 发送 |
| 管理接口 | 写 | 调度状态上报 | 周期性写入 | 向系统管理接口发送 |


## 安全边界

| 规则编号 | 内容 |
|:---:|------|
| R-01 | 本模块为 ECC 认知大脑的唯一对外网关，所有跨系统通信必须经本模块中转，禁止任何模块直连外部系统 |
| R-02 | 安全仲裁模块下发的熔断/降级/闭锁指令具有最高优先级，本模块必须无条件执行 |
| R-03 | 外部大模型调用必须经过配额管理，单日调用量不得超过预设限额，防止恶意或异常的高频调用 |
| R-04 | 所有跨系统通信必须记录完整的审计日志，包括请求来源、目标系统、数据摘要与时间戳 |
| R-05 | 主权闭锁状态一旦触发，仅可人工解除，任何自动化模块无权恢复自动化操作 |
| R-06 | 系统资源监控数据仅用于内部调度决策，不得泄露给外部系统 |
