# ag-ecc-06-结果评估模块 接口规格

---

## 基本信息

| 项 | 内容 |
|----|------|
| 模块编号 | ag-ecc-06 |
| 模块名称 | 结果评估模块 |
| 所属分区 | 一、认知大脑核心模块 |
| 核心职责 | 接收行动执行层（Agent-mcc-exec）返回的工具调用结果或任务步骤完成回执，对执行结果进行质量评估、完成度判定与异常识别。将评估后的结果反馈给 ag-ecc-02（任务规划模块）用于进度跟踪与异常恢复决策，同时将具有学习价值的成功/失败经验片段通过 ag-ecc-05（记忆查询模块）写入 MLNF-Mem 记忆中枢。不参与任务规划或工具执行，仅负责执行结果的客观评估与经验沉淀触发 |
| 依赖模块 | ag-ecc-02（任务规划模块，反馈步骤完成状态）、ag-ecc-05（记忆查询模块，将评估后的经验写入记忆中枢）、ag-mcc-exec（行动执行层，接收工具执行结果） |
| 被依赖模块 | ag-ecc-02（接收步骤完成回执与异常通知）、ag-ecc-08（元认知模块，可选地提供评估数据供自我反思） |


## 内部状态定义

| 状态 | 标识 | 含义 | 触发条件 |
|------|------|------|----------|
| 等待结果 | `WAITING_RESULT` | 无待评估结果，等待执行层返回 | 系统初始化完成，无活跃任务 |
| 评估中 | `EVALUATING` | 正在对执行结果进行多维度质量评估 | 收到执行层的工具调用结果或步骤回执 |
| 异常分析 | `ANOMALY_ANALYSIS` | 检测到执行异常，正在分析根因 | 执行结果状态为"失败"或"异常" |
| 经验写入 | `EXPERIENCE_WRITING` | 评估完成，正在将经验写入记忆中枢 | 评估完成且结果具有学习价值 |
| 暂停服务 | `SYSTEM_PAUSED` | 系统紧急熔断 | 收到紧急熔断指令 |


## 输入数据

| 输入项 | 数据类型 | 来源模块 | 触发条件 | 优先级 |
|--------|----------|----------|----------|:---:|
| 工具执行结果 | Struct（步骤ID + 计划ID + 工具名称 + 执行状态：成功/失败/超时 + 输出数据 + 耗时 + 错误码） | ag-mcc-exec 行动执行层（通过 CerebellumBus） | 每次工具调用完成时 | **高** |
| 步骤完成回执 | Struct（步骤ID + 计划ID + 完成状态 + 输出摘要 + 是否满足预期） | ag-mcc-exec 行动执行层 | 每个步骤执行完成时 | **高** |
| 预期结果定义 | Struct（步骤ID + 预期输出格式 + 可接受偏差范围 + 关键成功指标） | ag-ecc-02 任务规划模块 | 随步骤需求一并提供 | 普通 |
| 全局调度指令 | Enum（暂停/恢复/熔断） | ag-ecc-12 资源调度模块 | 模式切换或紧急事件时 | **紧急** |


## 输出数据

| 输出项 | 数据类型 | 目标模块 | 输出条件 | 优先级 |
|--------|----------|----------|----------|:---:|
| 步骤完成通知 | Struct（计划ID + 步骤ID + 评估结论：成功/需重试/失败不可恢复 + 偏差说明 + 建议操作） | ag-ecc-02 任务规划模块 | 评估完成时 | **高** |
| 异常中断通知 | Struct（计划ID + 失败步骤ID + 失败原因分类 + 建议恢复策略 + 已完成进度） | ag-ecc-02 任务规划模块 | 执行状态为"失败"或"异常"时 | **高** |
| 经验沉淀请求 | Struct（条目ID + 经验类型 + 场景标签 + 任务描述 + 工具序列 + 成功/失败标签 + 重要度预估值） | ag-ecc-05 记忆查询模块（转发至记忆中枢） | 评估完成且结果具有学习价值时 | 普通 |
| 评估状态上报 | Struct（当前状态 + 评估次数 + 成功率 + 平均耗时 + 常见失败原因分布） | ag-ecc-12 资源调度模块 | 周期性（每60秒）或状态变更时 | 普通 |


## 评估维度与判定规则

### 一、执行结果质量评估

| 评估维度 | 权重 | 判定方式 | 数据来源 |
|----------|:---:|------|------|
| 执行状态匹配度 | — | 状态=成功/失败/超时直接判定 | ag-mcc-exec 返回的状态字段 |
| 输出格式符合度 | 0.40 | 输出数据结构与预期格式的匹配程度 | 预期结果定义 vs 实际输出 |
| 执行耗时合理性 | 0.25 | 实际耗时与预估耗时的偏差比例 | 步骤需求中的预估耗时 vs 实际耗时 |
| 结果完整性 | 0.20 | 必要字段是否全部返回且非空 | 预期关键字段列表 vs 实际返回字段 |
| 异常信号检测 | 0.15 | 是否包含错误码、警告标记、空值等异常信号 | 输出数据中的异常标记 |

### 二、异常分类与处理建议

| 异常类型 | 判定条件 | 建议处理 |
|----------|------|------|
| 临时性错误 | 错误码为 TIMEOUT / NETWORK_ERROR / RATE_LIMITED | 自动重试（最多3次） |
| 参数错误 | 错误码为 INVALID_PARAM / MISSING_PARAM | 返回修正建议，等待用户补充参数后重试 |
| 权限不足 | 错误码为 PERMISSION_DENIED / UNAUTHORIZED | 立即终止，通知用户需授权 |
| 工具不可用 | 错误码为 SERVICE_UNAVAILABLE / TOOL_OFFLINE | 暂停该步骤，建议切换备选工具 |
| 未知错误 | 错误码不在已知分类中 | 标记为异常，触发元认知复盘 |


## 核心处理逻辑

```
FUNCTION result_evaluator_main_loop():
    STATE_WAIT = WAITING_RESULT
    STATE_EVAL = EVALUATING
    STATE_ANOMALY = ANOMALY_ANALYSIS
    STATE_WRITE = EXPERIENCE_WRITING
    STATE_PAUSED = SYSTEM_PAUSED

    SET internal_state = STATE_WAIT
    初始化统计计数器

    WHILE 系统运行中:
        // 第1步：紧急熔断
        IF 收到紧急熔断指令:
            SET internal_state = STATE_PAUSED
            CONTINUE
        ELSE IF 收到恢复指令 AND internal_state == STATE_PAUSED:
            SET internal_state = STATE_WAIT

        // 第2步：接收执行结果
        IF 收到执行结果:
            SET internal_state = STATE_EVAL
            结果 = 执行结果

            // 2a. 获取预期结果定义
            预期定义 = 向 ag-ecc-02 查询预期结果定义(结果.步骤ID)

            // 2b. 判定执行状态
            IF 结果.执行状态 == "成功":
                评估结论 = 评估成功结果(结果, 预期定义)
            ELSE:
                SET internal_state = STATE_ANOMALY
                评估结论 = 评估异常结果(结果)
                IF 评估结论.建议操作 == "自动重试":
                    向 ag-ecc-02 发送步骤完成通知(结果.步骤ID, "需重试", 评估结论)
                    CONTINUE

            // 2c. 发送评估结果至任务规划模块
            IF 评估结论.结论 == "成功":
                向 ag-ecc-02 发送步骤完成通知(结果.步骤ID, "成功", 评估结论)
            ELSE IF 评估结论.结论 == "失败不可恢复":
                向 ag-ecc-02 发送异常中断通知(结果.计划ID, 结果.步骤ID, 评估结论)

            // 第3步：经验沉淀判定
            SET internal_state = STATE_WRITE

            // 3a. 成功经验（具有高学习价值）
            IF 结果.执行状态 == "成功" AND 结果.耗时 > 历史平均耗时 * 1.5:
                经验条目 = 构建经验条目(结果, "成功优化", 重要度=0.55)

            // 3b. 失败经验（必须记录以供学习）
            IF 结果.执行状态 == "失败":
                经验条目 = 构建经验条目(结果, "失败", 重要度=0.65)

            IF 经验条目 != None:
                向 ag-ecc-05 发送经验沉淀请求(经验条目)

            SET internal_state = STATE_WAIT

        // 第4步：周期性状态上报
        IF 距上次状态上报 >= 60秒:
            向 ag-ecc-12 上报评估状态

        SLEEP 10ms


FUNCTION 评估成功结果(结果, 预期定义):
    偏差评分 = 计算输出格式符合度(结果.输出数据, 预期定义.预期输出格式)
    耗时评分 = 计算耗时合理性(结果.耗时, 预期定义.预估耗时)
    完整性评分 = 检查字段完整性(结果.输出数据, 预期定义.关键字段)

    IF 偏差评分 >= 0.8 AND 耗时评分 >= 0.6 AND 完整性评分 >= 0.9:
        RETURN {结论: "成功", 偏差说明: "输出符合预期"}
    ELSE:
        RETURN {结论: "成功但有偏差", 偏差说明: 生成偏差报告(偏差评分, 耗时评分, 完整性评分)}


FUNCTION 评估异常结果(结果):
    异常类型 = 分类错误码(结果.错误码)
    CASE 异常类型 OF:
        "临时性错误":
            RETURN {结论: "需重试", 建议操作: "自动重试"}
        "参数错误":
            RETURN {结论: "失败不可恢复", 建议操作: "修正参数后重试"}
        "权限不足":
            RETURN {结论: "失败不可恢复", 建议操作: "需用户授权"}
        "工具不可用":
            RETURN {结论: "需重试", 建议操作: "切换备选工具"}
        DEFAULT:
            RETURN {结论: "失败不可恢复", 建议操作: "人工介入"}
```


## 约束与异常处理

| 场景 | 处理方式 | 恢复条件 |
|------|----------|----------|
| 执行结果缺少必要字段（无状态码） | 标记为"结果格式异常"，通知任务规划模块暂停该计划 | 执行层返回格式正确的结果 |
| 预期结果定义查询失败（ag-ecc-02 无响应） | 使用宽松默认标准评估，标记"评估精度受限" | 查询恢复正常 |
| 经验写入失败（记忆中枢拒绝） | 缓存失败条目，定时重试（最多3次） | 记忆中枢恢复正常 |
| 连续3次评估为"需重试" | 自动升级为"失败不可恢复"，通知任务规划模块 | — |
| 紧急熔断 | 暂停评估，未完成的评估结果丢弃 | 紧急解除后恢复 |


## 总线契约

| 总线 | 操作 | 数据内容 | 权限 | 说明 |
|------|------|----------|------|------|
| CerebellumBus | 读 | 工具执行结果 / 步骤完成回执 | 只读 | ag-mcc-exec 发送 |
| MemoryBus | 读 | 预期结果定义 | 只读 | ag-ecc-02 返回 |
| MemoryBus | 写 | 步骤完成通知 | 专属写入 | 向 ag-ecc-02 发送 |
| MemoryBus | 写 | 异常中断通知 | 专属写入 | 向 ag-ecc-02 发送 |
| MemoryBus | 写 | 经验沉淀请求 | 专属写入 | 向 ag-ecc-05 发送（转发至记忆中枢） |
| MemoryBus | 写 | 评估状态上报 | 周期性写入 | 向 ag-ecc-12 发送 |


## 安全边界

| 规则编号 | 内容 |
|:---:|------|
| E-01 | 本模块仅评估执行结果，不参与任务规划、工具选择或安全仲裁决策 |
| E-02 | 写入记忆中枢的经验条目必须经过去个性化处理，不得包含用户的原始输入数据 |
| E-03 | 评估过程中使用的预期结果定义不得包含任何敏感信息（密钥、令牌等） |
| E-04 | 评估结论仅作为建议输出，任务规划模块有权根据全局态势进行最终决策 |


## 接口校验用例

| 用例编号 | 前置条件 | 输入 | 预期输出 |
|----------|----------|------|----------|
| TC-E06-01 | `WAITING_RESULT`，成功结果 | 执行结果（状态=成功，输出匹配预期） | 评估结论=成功，发送步骤完成通知，触发经验沉淀 |
| TC-E06-02 | `WAITING_RESULT`，临时性错误 | 执行结果（状态=失败，错误码=TIMEOUT） | 评估结论=需重试，建议操作=自动重试 |
| TC-E06-03 | `WAITING_RESULT`，权限不足 | 执行结果（状态=失败，错误码=PERMISSION_DENIED） | 评估结论=失败不可恢复，发送异常中断通知 |
| TC-E06-04 | `WAITING_RESULT`，成功但耗时异常 | 执行结果（状态=成功，耗时>历史均值1.5倍） | 标记为"成功但有偏差"，仍触发经验写入（学习价值） |
| TC-E06-05 | `EVALUATING`，连续3次需重试 | 同一步骤第3次返回临时性错误 | 自动升级为"失败不可恢复" |
| TC-E06-06 | `SYSTEM_PAUSED`，收到任何输入 | 任意执行结果 | 拒绝处理，保持暂停状态 |
