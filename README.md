## 一、仓库定位

本仓库为 EM-Core 通用智能系统中 ECC 认知大脑的 AI Agent 专项实现，固定 12 个顶层核心模块。ECC 为系统唯一上层认知推理中枢，负责意图解析、任务规划、风险校验、模型调度，无底层系统操作能力，所有可执行意图统一经 CerebellumBus 下发 MCC 执行。

ECC-12 是整套系统唯一双向出入口：入向接收用户输入、MCC 执行回执、MLNF 记忆反馈、LLM 返回报文；出向发起 LLM 请求、下发标准化执行意图、推送回复文本至前端。网关内置异步线程池与独立 session 输入队列，LLM 等待与 MCC 批量执行空档支持持续输入新指令。


## 二、核心架构

### 2.1 完整双向数据流闭环

1. 用户输入 → ECC-12 网关入队，分片预检高危命令
2. ECC-01 情境解析构建标准化情境结构体
3. ECC-02 意图规划拆解多步骤子任务，自动识别依赖关系生成任务树
4. ECC-12 按任务复杂度与用户偏好动态调度本地/云端 LLM
5. LLM 回流经 ECC-12 二次预检后存入 ECC-07 工作缓存
6. ECC-05 完整语义推演，生成 sign_ecc05 认知安全签名
7. 带签名意图报文经 CerebellumBus 下发 MCC，支持批量 DAG 并行调度
8. MCC 分步执行回执与故障回执持续回流 ECC-12，刷新 ECC-07 上下文
9. 当前 session 全部并发任务完成，ECC-12 解锁 MLNF 只读锁
10. 系统空闲时 ECC-05 批量审核 ECC-07 草稿，合规数据写入 MLNF 长效记忆

### 2.2 本地/云端双 LLM 动态调度

ECC-12 读取用户全局偏好，按任务复杂度自动分流。用户配置云端优先或高复杂任务走云端 API，配置本地优先或断网/重度降级走 ECC-11 本地轻量化推理引擎。两套通路共用同一套安全校验、上下文缓存、记忆锁、MCC 下发逻辑。

### 2.3 核心特性

| 特性 | 说明 |
|------|------|
| 认知执行完全解耦 | ECC 仅做推理规划，无系统操作能力，全部执行交由 MCC |
| ECC-12 双向无阻塞网关 | 异步线程池、session 独立队列，LLM/MCC 空档持续输入 |
| 双模型动态调度 | 出厂预设云端优先/本地优先，自动按任务复杂度分流，离线强制本地兜底 |
| 三层安全拦截 | 网关分片预检→ECC-05 语义推演签名→MCC 底层资源锁兜底 |
| session 记忆只读锁 | LLM 调用即下发锁，全会话并发任务全部成功后解锁 |
| 分级资源降级 | CPU>80%/内存>90%/电量<20% 限制并发，与 MCC 同步 |
| MCC 故障回执分发 | 接收标准化故障回执，按严重等级分流至 ECC-08/ECC-09 |
| 用户不满识别与响应 | ECC-10 识别 8 类不满信号，ECC-02 匹配响应策略，ECC-04 支持备选方案热切换 |
| 系统崩溃恢复 | ECC-09 结合 MLNF WAL 日志与 MCC-04 心跳重建未完成 session |
| Agent 间通信预留 | ECC-12 预留 AgentBus 协议接口 |


## 三、12 模块详细定义

### ECC-01 情境解析单元

解析用户原始输入、窗口快照、文件上下文、历史对话。提取实体、操作目标、约束条件，构建标准化情境结构体。长文本分片拆分，降低单次 LLM 推理负载。

### ECC-02 多步骤意图规划单元

结合 MLNF 历史记忆拆解用户需求为多步骤子任务。自动识别子任务依赖关系，生成适配 MCC DAG 并行调度的任务树，单次最大子任务数 12。标记任务风险等级与资源占用预估。用户不满响应策略子单元接收 ECC-10 不满信号，匹配对应响应策略。

### ECC-03 隐私内容检测单元

全链路扫描用户输入、LLM 输出、待下发 MCC 指令。识别身份证、密钥、银行卡、隐私路径，自动脱敏或直接拦截。

### ECC-04 全局风险规则引擎单元

内置永久黑名单：系统目录、磁盘格式化、全盘删除、内网渗透、勒索类操作。匹配后直接阻断推理流程，不生成认知签名。

### ECC-05 伦理仲裁与签名生成单元

完整语义推演，识别分段拼接、诱导式、隐藏式恶意脚本。风险分级采用 L0/L1/L2/L3 四级体系。校验通过生成全局唯一不可篡改 sign_ecc05 认知安全签名。系统空闲时批量审核 ECC-07 草稿，合规数据写入 MLNF 长效记忆。

### ECC-06 MLNF 记忆交互单元

通过 MemoryBus 向双漏斗记忆中枢发起相似度检索与历史经验查询。接收记忆返回的 S/V/C 量化指标。仅 ECC-05 审核通过且 session 锁解锁后推送新经验写入长效记忆。故障经验写入子单元将不可恢复故障经验写入漏斗二警示槽。

### ECC-07 会话工作缓存单元

按 session_id 隔离存储用户提问、LLM 返回草稿、MCC 分步执行回执。持久化临时草稿于本地临时分区，session 销毁后 24 小时自动物理清除。跨设备实时消息推送子单元向局域网内已配对同账户设备推送会话摘要。

### ECC-08 元认知自省单元

接收 MCC 全部并发任务执行成败与异常反馈。统计任务成功率与资源占用，修正后续意图规划策略。MCC 故障策略匹配子单元根据故障码匹配预设策略。

### ECC-09 全局执行状态机单元

统一管控全会话状态：Idle/LLM推理中/MCC执行中/Paused/Failed/Success。系统空闲判定条件：无活跃 LLM 请求 + MCC 执行队列为空 + 用户无操作超 5 分钟。空闲触发 ECC-05 批量草稿审核。系统崩溃恢复子单元恢复供电后自动检测未完成 session，向用户提供继续/重新开始/放弃选项。跨设备会话同步子单元查询局域网内其他设备活跃会话，支持无缝切换继续对话。

### ECC-10 反馈聚合与前端输出单元

聚合 MCC 多并行子任务执行日志与成败明细。整理自然语言汇总回复与异常提示。用户不满信号识别子单元精确分类 8 类不满信号，推送 ECC-02。

### ECC-11 本地轻量化推理引擎单元

内置轻量向量模型与文本分类模型，支撑离线问答与本地记忆匹配。重度资源降级下仅加载分类模型，保障基础离线功能可用。

### ECC-12 全域调度与双向网关单元

双向数据流管控：统一接收用户输入、MCC 回执、MLNF 反馈、LLM 回流；统一向外发起 LLM 请求、下发 MCC 意图、推送前端回复。分片高危预检子单元 Trie 分片实时拦截显性高危命令。多 session 并发调度子单元独立 session 输入缓冲区与异步多线程隔离。双 LLM 动态调度子单元按用户偏好与任务复杂度自动分流。session 记忆锁管控子单元 LLM 调用即下发锁，接收 MCC 完成回执后解锁。资源监控与分级降级子单元采集整机资源同步下发降级信号。熔断信号转发子单元终止指令或不可恢复故障时销毁排队请求并向 MCC 下发全域熔断。MCC 故障回执接收子单元按严重等级分流至 ECC-08 与 ECC-09。Agent 间通信总线预留子单元定义 AgentBus 标准报文格式。跨设备会话路由子单元接收其他设备会话恢复请求，传输 ECC-07 上下文。


## 四、全局锁定规范

| 规则编号 | 内容 |
|:---:|------|
| 1 | ECC 顶层模块固定 01–12 永久编号，新增能力均以内置子单元实现 |
| 2 | 认知、执行、记忆三层解耦架构、双向 ECC-12 网关、三层安全拦截链、双总线协议永久锁定 |
| 3 | 所有下发 MCC 执行意图必须携带 sign_ecc05，无签名 CerebellumBus 直接丢弃 |
| 4 | MLNF 长效记忆读写权限由 ECC-12 统一管控 session 只读锁，仅全会话所有并行任务成功后方可解锁 |
| 5 | ECC 无底层系统操作能力，所有原子操作仅可通过 CerebellumBus 下发 MCC 执行 |
| 6 | 资源降级机制与 MCC 完全同步，高负载同步限制并行推理与批量并发任务 |
| 7 | LLM 分段拼接恶意指令双层兜底：ECC-05 语义推演第一层，MCC 底层第二层拦截 |
